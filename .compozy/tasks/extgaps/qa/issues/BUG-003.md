# BUG-003: Missing test coverage for bundle validation in extension/bundle.go

**Severity:** High
**Priority:** P1
**Component:** `internal/extension/bundle.go`
**Related:** TC-FUNC-002

## Environment

- Code review of test files
- No dedicated test file exists for `bundle.go` validation logic

## Description

The bundle spec validation layer (`bundle.go`, 568 LOC) contains extensive validation logic but has zero dedicated unit tests. The validation functions cover:

1. `BundleSpec.Validate()` — bundle-level constraints
2. `BundleProfile.Validate()` — profile-level constraints
3. `BundleJob.Validate()` — job constraints including channel reference validation
4. `BundleTrigger.Validate()` — trigger constraints
5. `BundleBridgePreset.Validate()` — bridge constraints including secret slots
6. `LoadBundleSpecs()` — file loading, TOML/JSON parsing, dedup
7. `collectBundleFiles()` — file discovery and filtering
8. `bundleDocument.toBundleSpec()` — document merging with conflict detection

### Expected Behavior

Each validation function should have table-driven tests covering all error branches and valid cases.

### Actual Behavior

No test file `internal/extension/bundle_test.go` exists.

## Impact

- Validation regressions (e.g., accepting invalid bundles) won't be caught
- 20+ error paths are untested
- File loading logic (TOML/JSON parsing, path resolution) is unverified
- Fails the project's 80% coverage requirement for the extension package

## Suggested Fix

Create `internal/extension/bundle_test.go` with:
- TestBundleSpecValidation (table-driven: all error cases from TC-FUNC-002)
- TestBundleProfileValidation
- TestBundleJobValidation
- TestBundleTriggerValidation
- TestBundleBridgePresetValidation
- TestLoadBundleSpecs (using t.TempDir for TOML/JSON fixtures)
- TestCollectBundleFiles (directory walking, filtering)
