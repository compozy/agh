# BUG-004: Runtime E2E tests counted acpmock lifecycle diagnostics as prompt diagnostics

## Status

Fixed and re-verified.

## Severity

Medium.

## Scenario

`make test-e2e-runtime` failed after hosted MCP injection because the ACP mock driver now records a `session_new` lifecycle diagnostic containing the injected `agh-hosted-tools` MCP server before the first prompt diagnostic.

## Expected

Tests that assert prompt payloads should count prompt diagnostics only. Lifecycle diagnostics should remain available as evidence for hosted MCP injection without breaking prompt assertions.

## Actual

Two E2E tests expected exactly one diagnostics record and failed when the lifecycle record was present:

- `TestDaemonE2EMemoryRecallUsesCatalogSynthesisWithoutMutatingStoredUserMessage`
- `TestDaemonE2EMockAgentsRemainIsolated`

## Root Cause

The tests read the full acpmock diagnostics stream and assumed every record was a prompt execution. Hosted MCP session launch made that assumption invalid by adding a lifecycle `session_new` record.

## Fix

`internal/testutil/acpmock/diagnostics.go` now exposes `PromptDiagnostics`, and the affected E2E tests use it when asserting prompt-only behavior.

## Evidence

- Reproduction failure: `.compozy/tasks/tools-refac/qa/logs/final-test-e2e-runtime.log`
- Re-verified focused tests: `go test -race -count=1 -tags integration ./internal/daemon -run 'TestDaemonE2EMemoryRecallUsesCatalogSynthesisWithoutMutatingStoredUserMessage|TestDaemonE2EMockAgentsRemainIsolated'`
