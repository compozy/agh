# BUG-003: Automation native tools captured a nil manager during daemon boot

## Status

Fixed and re-verified.

## Severity

High.

## Scenario

Task 13 real-scenario QA enabled automation in the isolated daemon config and invoked:

```bash
agh tool info agh__automation_jobs_list -o json
agh tool invoke agh__automation_jobs_list --input '{}' -o json
```

## Expected

When `automation.enabled=true`, automation tool descriptors should be executable and route to the live automation manager.

## Actual

Automation tools stayed unavailable:

```text
error: tool_unavailable: tool "agh__automation_jobs_list" is unavailable
reason_codes: ["dependency_missing"]
```

## Root Cause

The daemon creates the native tool registry before booting the automation manager. `daemonNativeToolsDeps.Automation` captured the manager value at registry construction time, so it remained `nil` even after automation booted successfully.

## Fix

`internal/daemon/native_tools.go` now supplies automation through a dynamic runtime lookup while preserving direct injection for tests. Native automation handlers resolve the current manager at call time, so tools created before automation boot can become executable once the manager exists.
