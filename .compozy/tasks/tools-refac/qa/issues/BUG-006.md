# BUG-006: Runtime harness artifact test expected an unaugmented echo prompt

## Status

Fixed and re-verified.

## Severity

Medium.

## Scenario

`make test-e2e-runtime` failed in `TestStartRuntimeHarnessCapturesTranscriptAndEventsArtifacts`.

## Expected

The harness artifact test should prove the transcript artifact captures assistant output for the user prompt.

## Actual

The test required the exact substring `echo: hello harness`. The runtime now sends an augmented prompt containing `agh-situation-context` followed by the user message, so the echo agent correctly emitted `echo: <agh-situation-context>...hello harness`. The transcript was valid, but the assertion was stale.

## Root Cause

The test assumed prompt text is never augmented before provider dispatch.

## Fix

`internal/testutil/e2e/runtime_harness_integration_test.go` now asserts the artifact contains the echo marker and original user text, without requiring the runtime to omit context augmentation.

## Evidence

- Reproduction failure: `.compozy/tasks/tools-refac/qa/logs/final-test-e2e-runtime.log`
- Re-verified focused test: `go test -race -count=1 -tags integration ./internal/testutil/e2e -run TestStartRuntimeHarnessCapturesTranscriptAndEventsArtifacts`
