# ADR-008: Manifest-Authoritative Extension Tool Descriptors

## Status

Accepted

## Date

2026-04-28

## Context

Extension tools need both install-time auditability and runtime ergonomics. If descriptors are runtime-only, AGH cannot inspect policy, risk, schemas, or source grants before launching extension code. If descriptors are manifest-only, SDK helpers cannot verify that the running extension actually implements the declared handlers.

AGH already treats `extension.toml` as the install-time contract for resources, capabilities, Host API actions, subprocess launch, security grants, and bridge metadata. The Tool Registry should extend that model rather than create a second descriptor authority.

## Decision

Extension tool descriptors are manifest-authoritative with runtime reconciliation.

`extension.toml` declares every extension-host tool under `resources.tools`. Each declaration includes the canonical `id`, display metadata, input/output schemas, risk flags, toolsets, result budget, and handler binding.

The extension runtime may also expose descriptors through `provide_tools` or SDK metadata generated by `extension.tool(...)`, but those runtime descriptors are only a reconciliation signal. The daemon marks an extension-host tool executable only when the runtime descriptor matches the manifest-authoritative descriptor for identity, schema digests, risk flags, and handler binding.

Schema digests are lowercase hex `sha256` over RFC 8785 JCS-canonicalized JSON Schema subtrees. The digest excludes surrounding manifest metadata and treats `$ref` values literally. TypeScript SDK, Go SDK, and daemon validation must share digest byte-vector fixtures.

Runtime-only descriptors are rejected in the MVP. Manifest descriptors without a matching runtime handler remain operator-visible but unavailable with a deterministic reason code.

## Alternatives Considered

### Runtime-authoritative descriptors

- **Description**: The manifest only declares `tool.provider`; the extension reports all tools after initialize.
- **Pros**: Flexible and ergonomic for SDK authors.
- **Cons**: Requires executing extension code before AGH knows tool risk, schemas, or source policy.
- **Why rejected**: Install-time auditability is required for policy, docs, and operator diagnostics.

### Manifest-only descriptors

- **Description**: The manifest is the only descriptor source; runtime implementation is checked only when a call happens.
- **Pros**: Simple policy model.
- **Cons**: Handler mismatch is discovered too late, and SDK helper bugs are harder to detect.
- **Why rejected**: Runtime reconciliation catches drift before session exposure.

## Consequences

### Positive

- Operators and agents can inspect extension tool risk before extension code runs.
- SDK helpers can improve authoring ergonomics without becoming the source of truth.
- Tool install, enable, disable, conflict, and policy diagnostics remain deterministic.

### Negative

- Extension authors must keep manifest descriptors and SDK registration in sync.
- The daemon must compare runtime descriptors against manifest descriptors and report structured mismatch reasons.

### Risks

- Descriptor drift could make valid tools unavailable. Mitigation: SDK templates generate manifest snippets and tests assert manifest/runtime parity.
- Manifest fields could become verbose. Mitigation: SDK scaffolding and docs should generate the boilerplate.

## Implementation Notes

- Add `id`, `output_schema`, `risk`, `destructive`, `open_world`, `requires_interaction`, `max_result_bytes`, `toolsets`, `tags`, and `handler` fields to extension `ToolConfig`.
- Add `CapabilityToolProvider = "tool.provider"`, `ExtensionServiceMethodProvideTools = "provide_tools"`, and `ExtensionServiceMethodToolsCall = "tools/call"` to the subprocess protocol and capability-method map.
- Add deterministic mismatch reason codes such as `runtime_descriptor_missing`, `runtime_descriptor_mismatch`, and `handler_missing`.
- The TypeScript and Go SDKs must include tests that compare generated/runtime descriptors and JCS schema digests to manifest fixtures.
