## Shared Decisions

- **Package manager is Bun.** Treat all `pnpm --filter <pkg> <cmd>` references in task docs as `bun run --cwd <pkg> <cmd>`. Install with `bun add` (never edit `package.json` by hand).
- **`@agh/ui` triggers use `render` not `asChild`.** Every Base UI primitive exposed via `@agh/ui` (Button, DialogTrigger, SheetTrigger, PopoverTrigger, TooltipTrigger, DropdownMenuTrigger, CollapsibleTrigger, Item, …) cooperates with the Base UI `render={<element .../>}` pattern. There is no Radix-style `asChild` — callers that bring `<Button asChild>` or `<Item asChild>` compile in TS but render incorrectly. Use `render={<a />}` / `render={<RouterLink />}` instead.
- **`@agh/ui` Progress auto-renders its track.** `<Progress value>` internally mounts `<ProgressTrack><ProgressIndicator /></ProgressTrack>`. Consumers only pass `<ProgressLabel />` + `<ProgressValue />` as children; re-adding Track/Indicator double-renders the bar.
- **`@agh/ui` ToggleGroup is array-valued.** Base UI's ToggleGroup reads and emits `readonly string[]`, not a scalar. Use `defaultValue={["tasks"]}` / `value={[tasks]}` + `onValueChange={(groupValue: string[]) => …}` even for single-select. Scalar state silently no-ops at runtime.
- **`packages/ui` now has its own Vitest harness** (`packages/ui/vitest.config.ts` + `src/test-setup.ts`, included in root `vitest.config.ts` `projects`). New primitives should land with colocated `*.test.tsx` files rather than spinning up separate configs. The setup file also mocks `Element.prototype.scrollIntoView` for cmdk under jsdom.
- **Motion consumer hooks**: use `useReducedMotionConfig()` (context-aware) when asserting against a `MotionConfig` wrapper. Plain `useReducedMotion()` only reads `matchMedia` and ignores the provider.
- **Base UI + motion integration template**: for any Base UI primitive with a Portal/Popup lifecycle, wrap the Root in a controlled state component that passes `actionsRef={useRef()}` to Base UI and exposes `{ actionsRef, open }` via context. The Content renders `<AnimatePresence onExitComplete={() => actionsRef.current?.unmount()}>` around `{open && <Portal keepMounted>…</Portal>}`, and wires Backdrop/Popup via `render={<motion.div initial/animate/exit/>}`. This is the sanctioned pattern for motion-animated popups — do not re-introduce `data-open:animate-*` CSS keyframes alongside motion, it double-animates.
- **Select + Combobox kept CSS animations, not motion.** `SelectPortal` does not expose `keepMounted`, and Select's `alignItemWithTrigger` already suppresses the CSS animation; converting them to motion requires a deeper rewrite. They stay on `tw-animate-css` until a future task explicitly migrates them.
- **motion width animations ignore `MotionConfig reducedMotion`.** `reducedMotion="always"` zeros transform/opacity but not layout-affecting props like `width`. Any motion primitive that animates width (Sidebar collapse, future split-pane resize) must set `duration: useReducedMotionConfig() ? 0 : ...` explicitly.
- **Sidebar structural primitive contract (`@agh/ui`).** Slots: `rail | header | nav | footer`. Props: `collapsed | defaultCollapsed | onCollapse | panelWidth=240 | collapseBreakpoint=768 | collapseLabel`. The collapse trigger is built in (at the bottom of the rail) and emits `onCollapse(next)` + flips `aria-expanded`. Auto-collapse on narrow viewports is internal and does NOT fire `onCollapse`. Downstream host (task 13) should not wire its own collapse button.
- **AppSidebar host contract (task 13).** `web/src/components/app-sidebar.tsx` is a fully-controlled thin composition over `@agh/ui` `Sidebar`. Props: `collapsed` + `onCollapseChange(next: boolean)` (wired to `useSidebarStore.setCollapsed`), plus the domain slots (workspaces/active/agent tree/sessions/connectionStatus/health). Session state → StatusDot tone mapping used by the sidebar only: `active→success`, `starting→warning pulse`, `stopping→neutral pulse`, `stopped→neutral`. Header renders the `agh` wordmark + `ALPHA` chip (DESIGN.md §4 Site Header lockup). Tests render real `@agh/ui` primitives under `UIProvider reducedMotion="always"` and only mock `@tanstack/react-router`.
- **SplitPane structural primitive contract (`@agh/ui`).** Slots: `list | detail | detailEmpty`. Props: `listWidth=340 | onDetailClose | narrowBreakpoint=768 | backLabel="Back"`. When narrow + detail is present, the list hides and a Back button appears — hosts must pass `onDetailClose` to participate. AnimatePresence with `mode="wait"` drives the detail ↔ empty fade.
- **Base UI's `Input` is `Field.Control`.** Do not use it inside a `ComboboxPrimitive.Input render={}` — `Field.Control` intercepts the input state and the combobox stops receiving typed values in tests. Use a raw `<input>` with the primitive's classes instead.
- **Base UI Combobox input filtering under jsdom.** `userEvent.type` does not fire the input event the combobox listens to; use `fireEvent.change(input, { target: { value } })` to drive filter tests.
- **`web/src/lib/utils.ts` only re-exports `cn` from `@agh/ui`.** When a vitest test mocks `@agh/ui`, the factory must either (a) include a working `cn`, or (b) use `vi.mock("@agh/ui", async importActual => ({ ...(await importActual()), … }))`. Replacing the whole module leaves `cn` undefined and any downstream component that calls `cn(...)` crashes at render.
- **Base UI Menu group parts require `<Menu.Group>`.** `DropdownMenuLabel` (MenuGroupLabel) throws `MenuGroupRootContext is missing` if not wrapped in `DropdownMenuGroup`. Same rule applies to any group-scoped part.
- **Base UI's Accordion and ToggleGroup use `multiple` (boolean), not Radix's `type="single"/"multiple"`.** Tests and docs written against Radix conventions will not compile or will silently no-op.
- **`Pill` + `Pills` tone mapping (`@agh/ui`).** Legacy `design-system/pill` tones collapse onto the six-variant `@agh/ui` system via `web/src/lib/pill-variant.ts` (`pillVariantFromTone`). Mapping: `neutral → default`, `amber | accent → accent`, `green → success`, `violet → info`, `danger → danger`, `warning → warning`. `kind` is dropped (pills render at `size="sm"` for badges, `size="md"` for filter toggles); `emphasis` is dropped (tints give full semantic color). Downstream domain helpers (`taskStatusTone`, `bridgeStatusTone`, …) still return legacy strings — wrap them with `pillVariantFromTone()` at call sites instead of rewriting the formatters.
- **`Pills` is a tablist, not a radiogroup.** Renders `role="tab"` with `aria-selected`/`aria-pressed`/`data-active` on each item. Tests previously asserting `aria-checked` against `PillButton` must switch to `aria-selected`. The badge on selected items is available at `[data-slot="pills-badge"]`.
- **`@agh/ui` `Empty.icon` accepts both a Lucide component reference and a pre-rendered ReactNode.** Internal detection covers `typeof === "function"` (plain components) AND `{render, $$typeof}` (Lucide's memoized forwardRef shape). Do not assume `typeof Icon === "function"` is enough when accepting icon components in any new primitive.
- **oxfmt does not merge multiple single-import lines from the same module.** After scripted migrations that append `import { X } from "@agh/ui"` next to an existing `import { Y } from "@agh/ui"`, run a manual merge pass (not `make web-fmt`).
- **StatusDot / ConnectionIndicator tone vocabulary (`@agh/ui`).** DESIGN.md §4 tones only: `success | warning | danger | info | accent | neutral`. Legacy `amber/green/violet` strings are gone from both primitives. Domain helpers that still emit legacy tones (`automationSemanticTone`, …) must either switch to the matching `automationStatusTone` (new vocab) or wrap at the call site — `automationSemanticTone` itself stays in place for `AutomationTag`. `ConnectionIndicator` default labels are "Connected" / "Disconnected" / "Reconnecting" with `reconnecting` driving a `pulse=true` dot.
- **`@agh/ui` `Metric` has two detail slots.** `detail` = inline baseline-aligned mono micro-unit (matches mock `primitives.jsx`); `subtext` = Inter 13px secondary line (matches DESIGN.md §4 "Metric Cards With Subtext"). Legacy `MetricStrip detail="…long sentence…"` prose migrates to `subtext`, not `detail`. Value typography follows DESIGN.md (Inter 24px/700 -0.02em), not the mock's mono.
- **`@agh/ui` `Toaster` is `next-themes`-free.** The primitive accepts `theme` via `ToasterProps` (default `"system"`), letting Sonner's own `prefers-color-scheme` watcher track the app theme. `@agh/ui` also re-exports `toast` from `sonner`. Consumers mount `<Toaster />` once at the app root; do not wrap it in a next-themes adapter unless a manual theme override is required.
- **`ChatMessageBubble` role → shell mapping (`@agh/ui`).** State-free presentational shell. `role` drives layout: `user` right-aligns with a `surface-elevated` bubble (meta rendered above), `agent` left-aligns with no bubble (meta inline beside the agent name, secondary text body), `system` is a full-width tertiary-mono row flanked by `h-px flex-1` dividers, `tool`/`diff` are pass-through `flex-col` wrappers so callers drop a `ToolCallCard` or diff card inside. `align` prop override is supported (default derived from role). Slots: `chat-message | chat-message-inner | chat-message-meta | chat-message-body`. Prop `role` shadows the native ARIA `role`, so the interface uses `Omit<React.ComponentProps<"div">, "role">`.
- **`ToolCallCard` status → tone mapping (`@agh/ui`).** DESIGN.md §4 status-badge formula via `MonoBadge`: `running → accent RUNNING`, `done → success DONE`, `error → danger ERROR`. Slots: `tool-call-card | tool-call-card-header | tool-call-card-icon | tool-call-card-tool | tool-call-card-path | tool-call-card-status | tool-call-card-body`. The status badge lives in a `MonoBadge` with `data-slot="tool-call-card-status"` passed through — `MonoBadge` spreads `{...props}` after its default `data-slot`, so the override works cleanly and tests can select the badge without coupling to class names. No separate StatusBadge primitive yet (10px/600/0.08em vs MonoBadge 11px/500/0.06em delta is unresolved — follow-up if a future task wants pixel-exact status-badge typography).
- **`ToolCallCard` icon + name/path slot contract (`@agh/ui`, task 20).** `icon?: LucideIcon | React.ReactNode` — when omitted, defaults to TerminalIcon; when a component ref, rendered with the primitive's icon classes; when a pre-rendered ReactNode, rendered verbatim (caller owns `data-slot`). `toolName` + `filePath` accept any `React.ReactNode` so consumers can inject `data-testid`-bearing spans inline without bespoke testid props. Status badge selection stays `[data-slot="tool-call-card-status"]` + `data-tone` / `data-status` on the outer. Error status applies `data-[status=error]:border-[color:var(--color-danger)]/40` on the card outline.
- **Session MessageBubble role mapping (task 20).** `UIMessage.role` values flow into `ChatMessageBubble.role`: `user → user`, `assistant → agent`, `system → system`, `diff → diff`. Tool messages (`tool_call` / `tool_result`) never reach MessageBubble — they flow through `ToolGroupSection → ToolCallCard`.
