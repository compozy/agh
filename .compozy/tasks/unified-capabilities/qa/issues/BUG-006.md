# BUG-006: Web production build emitted an oversized shared chunk warning

**Severity:** Medium
**Priority:** P1
**Type:** Build Quality
**Status:** Fixed

## Environment

- **Build:** `feat/network-slack-redesign` after Round 1 commit `971c51cf`
- **OS:** Linux
- **Browser:** N/A
- **Surface:** Vite/Rolldown production build

## Summary

`make verify` passed functionally but the web production build emitted a Vite warning because the shared `src` chunk exceeded the default 500 kB threshold after minification. The repository guidance requires zero warnings in verification output, so the warning was treated as a blocking QA finding.

## Reproduction

```bash
bun run build:raw
```

Observed before the fix:

- Vite built successfully.
- The generated shared chunk was approximately `510.51 kB`.
- Vite emitted `Some chunks are larger than 500 kB after minification`.

## Expected

The production web build should complete without warning output while keeping real code splitting behavior explicit and maintainable.

## Root Cause

The default Rolldown chunking kept React runtime packages inside the shared app chunk. The app chunk was only slightly over the warning threshold, but enough to emit a build warning on every production build.

## Fix

Added an explicit Rolldown code-splitting group for React runtime packages in `web/vite.config.ts`. This extracts React, React DOM, Scheduler, and `use-sync-external-store` into a dedicated `react-runtime` chunk and keeps the shared app chunk below the warning threshold without raising or disabling the warning limit.

## Verification

```bash
bun run build:raw
make verify
go vet ./...
```

All commands passed after the fix. The final build produced `react-runtime` at approximately `191.21 kB` and the shared `src` chunk at approximately `497.69 kB`, with no Vite chunk-size warning.

## Impact

- **Users Affected:** CI/release verification and production asset loading.
- **Frequency:** Every production web build before the fix.
- **Workaround:** Raising `chunkSizeWarningLimit` would hide the issue, but was not used.
