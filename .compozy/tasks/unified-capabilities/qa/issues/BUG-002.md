# BUG-002: Same-daemon local trace lifecycle messages were dropped after capability handoff

**Severity:** High
**Priority:** P0
**Type:** Functional
**Status:** Fixed

## Environment

- **Build:** `a572694e`
- **OS:** Darwin 25.3.0
- **Browser:** N/A
- **URL:** Local daemon/network lifecycle flow

## Summary

Directed same-daemon capability interactions could open successfully and then drop later `receipt` or `trace` lifecycle messages. The sender-side interaction map was marked terminal before the local receive path processed the message, so the receiver treated the update as already closed and ignored it.

## Reproduction

```bash
go test -v ./internal/network -count=1 -run 'TestManagerDeliversLocalTraceLifecycleMessages'
```

Observed before the fix:

- Same-daemon local `receipt` and `trace` messages on an existing capability interaction were ignored.
- Operator-visible collaboration state lost terminal lifecycle progress for local interactions.

## Expected

Local-directed `kind:"capability"` interactions should accept later `receipt` and `trace` messages on the same interaction until the receive path closes the interaction according to normal lifecycle rules.

## Root cause

`syncSentLifecycle` updated sender-side interaction state for local-directed `receipt` and `trace` envelopes before the same router instance processed them as inbound local messages.

## Fix

Added `shouldSyncSentLifecycle` and skipped outbound pre-sync for local-directed `receipt` and `trace` envelopes. Added a focused regression that proves same-daemon local trace delivery still advances the interaction lifecycle.
