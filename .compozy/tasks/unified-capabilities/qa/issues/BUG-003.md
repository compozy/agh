# BUG-003: Network operator browser flow was unstable across reload and dialog interactions

**Severity:** Medium
**Priority:** P1
**Type:** UI
**Status:** Fixed

## Environment

- **Build:** `a572694e`
- **OS:** Darwin 25.3.0
- **Browser:** Chromium via Playwright
- **URL:** `/network`

## Summary

The shipped network operator flow was not stable enough for the real browser lane. Route remounting used a stale pathname source, and the dialog popup animation scaled the click target during interaction. In practice, the network flow could lose continuity across reloads or become flaky when creating a channel through the operator UI.

## Reproduction

```bash
bun run --cwd web test:e2e:daemon-served:raw e2e/network.spec.ts
```

Observed before the fix:

- The operator flow could lose route continuity after reload.
- Channel-creation dialog interactions were unstable enough to fail the real browser scenario.

## Expected

The network operator browser flow should create a channel, inspect peers, observe timeline state, and reload without losing route continuity or click stability.

## Root cause

`AppLayout` keyed route state from a stale pathname source, and the dialog popup/overlay animation used scale transforms that changed pointer geometry during the interaction.

## Fix

Updated `AppLayout` to key from `router.latestLocation.pathname || locationPathname`, memoized dialog popup/overlay elements, and switched popup animation to fade-only so browser interaction geometry stays stable. Added targeted route and dialog regression coverage alongside the browser lane.
