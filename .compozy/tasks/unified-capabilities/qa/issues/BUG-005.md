# BUG-005: Network browser QA targeted removed tab/list selectors

**Severity:** Medium
**Priority:** P1
**Type:** Test Automation
**Status:** Fixed

## Environment

- **Build:** `feat/network-slack-redesign` after Round 1 commit `971c51cf`
- **OS:** Linux
- **Browser:** Chromium via Playwright
- **Surface:** `web/e2e/network.spec.ts`

## Summary

The network Playwright spec and browser artifact collector still targeted the pre-redesign tab/list/detail test IDs, including `network-tab-channels`, `network-channels-list-panel`, and `network-channel-detail-panel`. The current UI uses the unified room workspace and exposes room-oriented test IDs such as `network-workspace`, `network-room-channel-*`, `network-room-peer-*`, `network-message-*`, and `network-details-panel`.

## Reproduction

```bash
bunx playwright test e2e/network.spec.ts e2e/settings.spec.ts --reporter=list
```

Observed before the fix:

- Settings browser coverage passed.
- The network spec failed while waiting for the removed `network-tab-channels` locator.

## Expected

Browser QA should exercise the current unified network workspace using stable selectors that match the room-based UI model and should keep route-state artifact collection aligned with that model.

## Root Cause

The network UI was redesigned from tabbed channel/peer panels into a unified room sidebar and detail workspace, but the E2E selector fixture, route-state collector, and network spec were not updated with the new test IDs and interaction model.

## Fix

Updated the E2E selector fixture, selector unit tests, browser artifact route-state collector, collector unit tests, and the network Playwright spec to use the current unified room selectors. The browser spec now verifies the create-channel dialog shell, seeds the channel through the public HTTP API for deterministic daemon-served setup, and asserts the real UI room list, details panel, message feed, direct peer room, and reload behavior.

The direct create-channel dialog submit path remains covered by focused Vitest route/component tests; the browser test keeps the end-to-end assertion on the daemon-served network workspace after deterministic API setup.

```bash
bun run test:raw src/routes/_app/-network.test.tsx src/routes/_app/settings/-network.test.tsx src/systems/network/components/network-create-channel-dialog.test.tsx src/systems/network/lib/network-formatters.test.ts

```

All commands passed after the fix.

## Impact

- **Users Affected:** QA automation and browser evidence generation for the network redesign.
- **Frequency:** Always for the network Playwright spec before the fix.
- **Workaround:** None; the spec had to be updated to current UI semantics.
