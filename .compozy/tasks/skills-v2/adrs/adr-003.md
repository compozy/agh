# ADR-003: Pluggable Registry Interface for Marketplace

- **Status:** Accepted
- **Date:** 2026-04-07
- **Context:** The skills marketplace needs a backend for search, download, and verification. Multiple registries exist (ClawHub, GitHub, potential custom registries).

## Decision

Define a `Registry` interface in `internal/skills/marketplace/` with `Search`, `Download`, and `Info` methods. Ship a ClawHub implementation as the default. The interface allows future registry backends without modifying the skills package or CLI commands.

Registry selection via config: `[skills.marketplace] registry = "clawhub"` (default).

## Alternatives Considered

1. **ClawHub-only hardcoded** — ships faster but creates vendor lock-in and requires code changes to support other registries.
2. **Git-based (no API)** — `agh skill install github.com/author/skill` directly from Git repos. No centralized search/discovery, poor UX for browsing available skills.

## Consequences

- New `internal/skills/marketplace/` package with `Registry` interface
- `internal/skills/marketplace/clawhub/` package for ClawHub implementation
- CLI commands (`skill search`, `skill install`) dispatch through the interface
- Adding a new registry = implement the interface + register in config
- HTTP client with exponential backoff, context-aware cancellation
