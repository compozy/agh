# Refactoring Analysis: Consolidated Summary

> **Date**: 2026-04-06
> **Scope**: Full `internal/` codebase -- 15 packages, ~30K LOC production code
> **Reports**: 5 individual analyses covering all packages
> **Methodology**: Martin Fowler's code smells catalog + SOLID analysis

---

## Overall Health

The AGH codebase is **well-architected for a greenfield alpha** -- clean interface boundaries, proper dependency inversion, no circular dependencies, disciplined concurrency patterns, and good test coverage. The dependency graph flows strictly downward as designed: `daemon/` -> API/CLI -> domain -> infrastructure.

The dominant issue is **cross-package and cross-file duplication**, accounting for ~50% of all findings. This is expected in an alpha with rapid feature development where patterns emerged independently before consolidation. The second pattern is **file bloat** -- five files exceed 1,000 lines and would benefit from file-level splitting (no API changes needed).

## Cross-Cutting Themes

### 1. Massive httpapi/udsapi Duplication (P0 -- ~900 lines production, ~1,700 lines tests)

The single biggest finding across the entire codebase. `httpapi` and `udsapi` share ~90% of their logic -- handlers, payload structs, SSE infrastructure, query parsers, conversion functions, memory handlers, workspace handlers, and test stubs. The `apisupport` package was the right instinct but covers only ~10% of the duplicated surface.

**Recommendation**: Extract a shared `internal/apicore/` package with payloads, conversions, parsers, SSE, memory handlers, workspace handlers, and a `BaseHandlers` struct. Reduce both transport packages to thin transport-binding layers.

**Report**: [API Layer](./20260406-api-layer.md) -- F1

### 2. Five 1,000+ Line Files Need File-Level Splitting

| File | Lines | Package | Responsibilities |
|------|-------|---------|-----------------|
| `daemon/daemon.go` | 1,495 | daemon | boot, dream, orphan cleanup, boundary checks, process utils |
| `session/manager.go` | 1,205 | session | lifecycle, workspace, prompting, recording, cleanup |
| `store/global_db.go` | 1,097 | store | workspace CRUD, session registry, observability, permissions |
| `workspace/resolver.go` | 1,078 | workspace | CRUD, resolution, caching, scanning, cloning, ID gen |
| `udsapi/handlers.go` | 1,058 | udsapi | all handler methods, payload types, SSE, parsers, conversions |

**Recommendation**: File-level splits within each package (no API changes). Each file already has clear responsibility clusters that can be separated.

**Reports**: [Core](./20260406-core-session-acp.md) -- F1, [Storage](./20260406-storage-observe-memory.md) -- F1, [Infra](./20260406-config-daemon-cli.md) -- F3, [New](./20260406-skills-workspace.md) -- F2, [API](./20260406-api-layer.md) -- F2

### 3. Duplicated Utility Functions Across Packages

Functions that appear in 2+ packages with identical or near-identical implementations:

| Utility | Locations | Lines |
|---------|-----------|-------|
| `processAlive` / `signalProcess` | daemon, cli, memory | ~60 |
| `userAgentsSkillsDir` | daemon, cli | ~40 |
| `atomicWriteFile` | store, memory | ~60 |
| `cloneRawMessage` / `cloneRawJSON` | session, acp | ~16 |
| `firstNonEmpty` | session, cli | ~12 |
| `fileSnapshot` + `snapshotsEqual` + clone | skills, workspace | ~60 |
| `testContext(t)` | 7 packages (store, observe, memory, acp, session, daemon, cli) | ~35 |
| `equalStringSlices` | store, observe, daemon | ~30 |
| `checkContext` | skills, workspace | ~10 |
| `normalizeAbsolutePath` / `expandUserPath` | daemon, config | ~40 |
| **Total duplicated** | | **~363 lines** |

**Recommendation**: Extract to shared internal packages:
- `internal/procutil/` -- process alive, signal
- `internal/fileutil/` -- atomic write
- `internal/fsnap/` -- file snapshots (or put in `workspace/`)
- `internal/testutil/` -- test context, string slice comparison
- Consolidate string/JSON helpers within consuming packages

**Reports**: [Core](./20260406-core-session-acp.md) -- F5, [Storage](./20260406-storage-observe-memory.md) -- F3/F7, [Infra](./20260406-config-daemon-cli.md) -- F1/F2/F4, [New](./20260406-skills-workspace.md) -- F1

### 4. Validation Boilerplate

13 `Validate()` methods in `store/` and additional validators in `config/` share identical patterns: `strings.TrimSpace(x) == ""` checks and `if q.Limit < 0` guards. ~80+ lines of mechanical repetition.

**Recommendation**: Extract `requireField(value, label)` and `requirePositiveLimit(limit, label)` helpers.

**Report**: [Storage](./20260406-storage-observe-memory.md) -- F2

### 5. Nil-Receiver Guard Repetition

18+ methods in `global_db.go` and 10+ methods in `session.go` begin with identical nil-receiver + nil-context guards.

**Recommendation**: Extract `checkReady(ctx)` method on receiver types.

**Reports**: [Storage](./20260406-storage-observe-memory.md) -- F6, [Core](./20260406-core-session-acp.md) -- F14

## Recommended Refactoring Roadmap

### Phase 1: Quick Wins (trivial effort, high clarity -- ~2-3 hours total)

These are safe, mechanical changes with zero API impact:

1. Extract `processAlive` + `signalProcess` to `internal/procutil/`
2. Extract `atomicWriteFile` to `internal/fileutil/` (fixes subtle Sync divergence)
3. Extract `testContext` + `equalStringSlices` to `internal/testutil/`
4. Consolidate `userAgentsSkillsDir` into `config/home.go`
5. Merge `cleanupFailedCreate` / `cleanupFailedResume` into `cleanupFailedStart(sessionDir, ...)`
6. Extract `processSkill` method in skills registry (3x loop dedup)
7. Replace `reflect.DeepEqual` with snapshot comparison in skills registry
8. Merge `startingDaemonStatus` / `stoppedDaemonStatus` into parameterized function
9. Fix typo `defaultReadHeaderTimout` in `udsapi/server.go`
10. Remove custom `max()` in `cli/format.go` (use Go builtin)

### Phase 2: File-Level Splits (moderate effort, zero API risk -- ~4-5 hours total)

All are file-level reorganizations within the same package:

1. Split `daemon/daemon.go` -> `daemon.go`, `boot.go`, `dream.go`, `orphan.go`, `boundary.go`
2. Split `session/manager.go` -> `manager.go`, `manager_lifecycle.go`, `manager_prompt.go`, `manager_workspace.go`
3. Split `store/global_db.go` -> `global_db.go`, `global_db_workspace.go`, `global_db_session.go`, `global_db_observe.go`
4. Split `workspace/resolver.go` -> `resolver.go`, `resolver_crud.go`, `scanner.go`, `clone.go`, `helpers.go`
5. Split `udsapi/handlers.go` -> `sessions.go`, `agents.go`, `observe.go`, `stream.go`, `payloads.go` (matching httpapi layout)
6. Split `store/schema.go` -> `schema.go`, `sqlite.go`, `migrate_workspace.go`

### Phase 3: Structural Deduplication (significant effort, high impact -- ~1-2 days)

1. Extract shared `internal/apicore/` package from httpapi/udsapi
   - Payloads, conversions, parsers, SSE, memory handlers, workspace handlers
   - Shared interfaces, `BaseHandlers` struct
   - Keep transport-specific logic in each package (CORS, socket lifecycle, error masking)
2. Consolidate test infrastructure into `internal/apitest/`
3. Extract `activateAndWatch` method to deduplicate Create/Resume
4. Extract `emitPermissionEvent` in ACP handlers
5. Extract validation helpers (`requireField`, `requirePositiveLimit`) in store
6. Extract shared `fileSnapshot` type across skills/workspace

### Phase 4: Deeper Improvements (when opportunistic)

1. Evaluate and remove legacy transcript parsers (zero-compat policy)
2. Make `timeNowUTC` injectable in `acp/` for test determinism
3. Extract generic `listBundle[T]` in CLI for output bundle boilerplate
4. Consider extracting `promptState` and `permissionBroker` from `AgentProcess`
5. Add `cloneConfig` completeness test using reflection

## Architectural Observations

### What's Working Well

- **No circular dependencies** -- strict downward dependency flow across all 15 packages
- **Interfaces defined where consumed** -- `AgentDriver`, `EventRecorder`, `WorkspaceResolver` all follow Go idioms
- **`daemon/` as sole composition root** -- properly imports all packages, no back-pointers
- **Functional options pattern** -- consistent across Manager, Observer, Resolver, Registry
- **Concurrency discipline** -- proper `context.Context` propagation, mutex usage, goroutine lifecycle management
- **Good test coverage** -- table-driven tests, `-race` flag, integration tests with real dependencies

### Areas to Watch

- **`workspace` package has 11 dependents** -- highest afferent coupling in the system. Changes to exported types (`Workspace`, `ResolvedWorkspace`) ripple widely. Currently stable, but worth monitoring.
- **`config` package depended on by ~8 packages** -- expected for configuration, but changes to `Config` struct require updating overlay types in `merge.go` (3 files per field).
- **`udsapi.SessionManager` missing `ApprovePermission`** -- functional gap vs. httpapi's interface. Should be resolved during apicore extraction.
