# ADR-002: Custom Sidebar with Workspace Icon Rail

## Status

Accepted

## Date

2026-04-09

## Context

The Paper design specifies a two-zone sidebar layout:
1. **Workspace icon rail** (40px wide): circle avatars with single letters (App logo with accent bg, active workspace with accent ring, inactive workspaces with surface bg). Always visible.
2. **Sidebar panel** (~220px): workspace name, search, agent list with session counts, workspace nav (Knowledge, Skills), system footer (connection status, version, settings). Collapsible.

The current app uses shadcn's `Sidebar` component which provides a single-panel sidebar with `collapsible="none"`. It does not support an icon rail separate from the main panel.

## Decision

Replace the shadcn Sidebar with a custom two-zone sidebar component:

1. Create `app-sidebar.tsx` with a fixed icon rail (40px) + collapsible panel (~220px)
2. Use standard HTML/CSS flexbox layout, not shadcn Sidebar primitives for the outer shell
3. Continue using shadcn primitives (buttons, menus, tooltips) for interactive elements inside the sidebar
4. Support collapsed mode (icon rail only, 40px) and expanded mode (icon rail + panel, ~260px)

## Alternatives Considered

### Alternative 1: Extend shadcn Sidebar

- **Description**: Keep shadcn Sidebar as base, add a custom icon rail zone adjacent to it
- **Pros**: Less code to write, maintains shadcn conventions
- **Cons**: shadcn Sidebar has strong opinions on layout, collapse behavior, and width management that conflict with the icon rail pattern. Fighting the component's internals.
- **Why rejected**: More effort to work around shadcn constraints than to build a clean custom solution

### Alternative 2: Style-only adjustments to shadcn Sidebar

- **Description**: Approximate the Paper design by styling the existing shadcn Sidebar
- **Pros**: Minimal code changes
- **Cons**: Cannot achieve icon rail + panel layout, no workspace switching avatars, fundamentally different structure
- **Why rejected**: Cannot match Paper design — visual fidelity is a requirement

## Consequences

### Positive

- 1:1 match with Paper design including icon rail, collapse behavior, and workspace switching
- Full control over collapse animation (200ms ease-in-out per DESIGN.md)
- Clean separation of workspace selection (rail) from workspace content (panel)

### Negative

- Must implement sidebar accessibility (focus management, keyboard nav) manually
- Lose shadcn Sidebar's built-in mobile responsive behavior

### Risks

- **Risk**: Accessibility gaps — Mitigation: use `role="navigation"`, `aria-label`, keyboard trap for collapsed mode
- **Risk**: Mobile layout — Mitigation: on narrow viewports, hide icon rail, show sidebar as overlay/sheet

## Implementation Notes

- Icon rail uses `nav` element with `aria-label="Workspaces"`
- Panel uses `aside` element with `aria-label="Sidebar"`
- Collapse state managed via Zustand store (shared with header for toggle button)
- Active workspace accent ring: `2px solid #E8572A` with 2px spacing
- App logo: `#E8572A` background, white letter
