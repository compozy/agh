# BUG-002: QA bootstrap can emit overlong Unix socket paths

**Severity:** High
**Priority:** P0
**Type:** Functional
**Status:** Fixed

## Environment

- **Build:** local `tools-registry` worktree, Task 16 QA pass
- **OS:** macOS
- **Browser:** N/A
- **URL:** `.agents/skills/agh-qa-bootstrap/scripts/bootstrap-qa-env.py`

## Summary

The QA bootstrap helper placed `AGH_HOME`, `AGH_UDS_PATH`, tmux socket paths, and the provider `HOME` under the long artifact workspace by default. On macOS, the generated Unix socket paths exceeded the portable 103-byte limit, making the fresh lab invalid for daemon and UDS validation.

## Reproduction

```bash
python3 .agents/skills/agh-qa-bootstrap/scripts/bootstrap-qa-env.py \
  --scenario tools-registry-task16 \
  --repo-root .
```

Observed before the fix:

- Initial manifest emitted `AGH_UDS_PATH=/Users/pedronauck/dev/qa-labs/agh-tools-registry-task16-20260429-073832-489478-lab/.agh/runtime/aghd.sock` at 105 bytes.
- The default provider-home daemon socket path was 114 bytes.
- UDS-dependent verification failed before Task 16 smoke execution.

## Expected

Fresh QA bootstrap manifests must produce unique runtime/provider paths whose daemon, tmux, and provider-default Unix socket paths are within the portable socket limit.

## Root cause

The bootstrap helper only guaranteed workspace isolation. It did not validate Unix socket path lengths and always derived runtime/provider paths from the long artifact workspace path.

## Fix

- Added portable Unix socket path length validation to the bootstrap helper and manifest health checks.
- When workspace-derived runtime paths exceed the limit, the helper now allocates unique short runtime/provider homes under the system temp directory while keeping QA artifacts in the workspace lab.
- Rebuilt the Task 16 lab and mirrored the corrected manifest into `.compozy/tasks/tools-registry/qa/bootstrap-manifest.json`.

## Verification

- `python3 -m py_compile .agents/skills/agh-qa-bootstrap/scripts/bootstrap-qa-env.py`
- `python3 .agents/skills/agh-qa-bootstrap/scripts/bootstrap-qa-env.py --scenario tools-registry-task16 --repo-root .`
- Evidence log: `.compozy/tasks/tools-registry/qa/logs/bootstrap-rerun-short-runtime.log`
- Corrected manifest paths:
  - `AGH_UDS_PATH` length: 85 bytes
  - `TMUX_BRIDGE_SOCKET` length: 92 bytes
  - provider default daemon socket length: 93 bytes

## Impact

- **Users Affected:** AGH QA passes on systems with strict Unix socket path limits, especially macOS.
- **Frequency:** Always when the generated lab workspace path is long enough to push socket paths beyond the limit.
- **Workaround:** Manually supplying a short workspace root could avoid the symptom, but the bootstrap helper must produce a valid lab by default.

## Automation Follow-up

- **Required:** Yes
- **Status:** Added
- **Spec / Command:** bootstrap helper path-length validation and manifest health validation
- **Notes:** Reuse of an existing manifest now fails health checks when socket paths exceed the limit.
