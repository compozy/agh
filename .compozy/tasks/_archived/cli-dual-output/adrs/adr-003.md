# ADR-003: Incremental Migration with Toon Fallback

## Status

Accepted

## Date

2026-03-31

## Context

The codebase has ~45 call sites of `writeCommandOutput` across 15+ CLI files, each calling `toon.Render*` functions. Migrating all commands to dual output at once is risky and blocks delivery. We need a strategy that allows incremental adoption while keeping the system functional at every step.

## Decision

Implement a `writeOutput` dispatcher function that accepts both a human renderer and a toon renderer. Commands that haven't been migrated yet pass `nil` for the human renderer, and the dispatcher falls back to toon output regardless of the resolved format. Migration proceeds command-by-command without breaking anything.

## Alternatives Considered

### Alternative 1: Big Bang Migration

- **Description**: Migrate all ~45 call sites at once in a single PR
- **Pros**: One-shot, no intermediate state
- **Cons**: High risk, large PR, hard to review, blocks all other CLI work
- **Why rejected**: Too risky for a change touching every CLI command file.

### Alternative 2: Feature Flag per Command

- **Description**: Add a per-command feature flag to enable/disable human output
- **Pros**: Granular control over rollout
- **Cons**: Over-engineered, adds configuration complexity, flags need cleanup later
- **Why rejected**: YAGNI. The nil-fallback approach achieves the same result with zero configuration.

## Consequences

### Positive

- Zero breaking changes at any point during migration
- Each command can be migrated and tested independently
- PRs are small and reviewable
- Unmigrated commands work exactly as they do today

### Negative

- During migration, some commands will have human output and others won't — inconsistent UX temporarily
- Need to track which commands are migrated (a simple checklist suffices)

### Risks

- Risk: migration stalls and some commands never get human renderers → Mitigation: prioritize high-traffic human commands first (status, ps, session list)

## Implementation Notes

```go
func writeOutput(cmd *cobra.Command, format string, humanFn, toonFn func() (string, error)) error {
    if format == "human" && humanFn != nil {
        rendered, err := humanFn()
        // ...
    }
    // fallback to toon
    rendered, err := toonFn()
    // ...
}
```

- Phase 1: Infrastructure (output.go, flag, format resolution, logger)
- Phase 2: High-traffic commands (status, ps, session list, session status)
- Phase 3: Remaining commands
