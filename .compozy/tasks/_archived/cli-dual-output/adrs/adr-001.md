# ADR-001: Environment Variable Detection for Output Format Default

## Status

Accepted

## Date

2026-03-31

## Context

AGH serves two audiences: humans running CLI commands interactively, and AI agents spawned by the kernel. Currently all output uses TOON format, which is optimized for agent parsing but poor for human readability. We need a mechanism to automatically select the right output format for each audience without requiring manual flags on every invocation.

Common approaches include TTY detection (`isatty`), explicit flags, or environment variable detection. TTY detection introduces edge cases with Docker `-t`, SSH sessions, CI runners, and IDE terminals. The AGH kernel already sets `COLLAB_AGENT` and `AGI_AGENT` environment variables on every spawned agent process.

## Decision

Use the presence of `COLLAB_AGENT` or `AGI_AGENT` environment variables as the primary mechanism to detect agent context and default the output format to `toon`. When these variables are absent, default to `human`. An explicit `-o`/`--output` flag overrides all automatic detection.

Priority chain: **explicit flag > env var detection > default "human"**.

## Alternatives Considered

### Alternative 1: TTY Detection

- **Description**: Use `isatty.IsTerminal(os.Stdout.Fd())` to detect terminal vs pipe
- **Pros**: Industry standard (used by gh, docker, kubectl), handles piping automatically
- **Cons**: Edge cases with Docker `-t`, SSH, CI runners, IDE terminals; non-deterministic output
- **Why rejected**: Introduces ambiguity in environments we don't control. AGH agents may run in TTY-attached sessions. The env var approach is deterministic because we control both sides.

### Alternative 2: Prompt Template Instructions Only

- **Description**: Instruct agents via prompt templates to always pass `-o toon`
- **Pros**: Zero code change, works immediately
- **Cons**: Depends on LLM compliance, agents may forget, fragile
- **Why rejected**: Non-deterministic. LLMs can and do ignore instructions.

### Alternative 3: TTY Detection + Flag Override

- **Description**: Combine isatty with an explicit flag
- **Pros**: More automatic than env var for non-AGH contexts
- **Cons**: Same TTY edge cases, more complex resolution logic
- **Why rejected**: Over-engineering for the current use case. AGH's two audiences are well-separated by env vars.

## Consequences

### Positive

- Deterministic output: same command always produces same format in same context
- Zero friction for agents: kernel already sets the env vars, no template changes needed
- Zero friction for humans: default is human-friendly, no flags needed
- Simple override: `-o toon` or `-o human` when needed

### Negative

- If someone sets `COLLAB_AGENT` manually outside AGH, they get TOON unexpectedly (unlikely edge case, easily overridden with `-o`)
- Non-AGH automation (scripts) that don't set these env vars get human output by default (use `-o toon` explicitly)

### Risks

- Risk: future integrations may need programmatic output without env vars → Mitigation: `-o` flag is always available as escape hatch

## Implementation Notes

- Resolution logic lives in `internal/cli/output.go` as `resolveOutputFormat(cmd)`
- Checks `cmd.Flag("output").Changed` first, then `os.Getenv("COLLAB_AGENT")` / `os.Getenv("AGI_AGENT")`
- Flag registered as persistent on root command so all subcommands inherit it
