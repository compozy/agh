# BUG-004: Zero handler tests for all 8 bundle API endpoints

**Severity:** High
**Priority:** P1
**Component:** `internal/api/httpapi/handlers_test.go`, `internal/api/udsapi/handlers_test.go`
**Related:** TC-INT-001 through TC-INT-010

## Environment

- Code review of test files
- Only route existence tests exist; no handler behavior tests

## Description

All 8 bundle API endpoints have zero handler tests. The existing test (`TestRegisterRoutesCoversTechSpecEndpoints`) only verifies that routes are registered, not that handlers:

1. Correctly bind and validate request bodies
2. Return correct HTTP status codes
3. Return correct response payloads
4. Handle error cases (nil service, invalid input, not found, conflicts)
5. Map service errors to correct HTTP statuses

### Untested Endpoints

| Endpoint                      | Method | Test Exists |
| ----------------------------- | ------ | ----------- |
| /api/bundles/catalog          | GET    | No          |
| /api/bundles/preview          | POST   | No          |
| /api/bundles/activations      | GET    | No          |
| /api/bundles/activations      | POST   | No          |
| /api/bundles/activations/:id  | GET    | No          |
| /api/bundles/activations/:id  | PATCH  | No          |
| /api/bundles/activations/:id  | DELETE | No          |
| /api/bundles/network/settings | GET    | No          |

## Impact

- API contract violations won't be caught
- Error mapping regressions (wrong HTTP status codes)
- Request validation gaps (malformed JSON, missing fields)
- Response payload structure changes undetected
- Both HTTP and UDS transports affected

## Suggested Fix

Add handler tests following the project's existing `apitest` helper pattern:

- Mock the bundle service interface
- Test each endpoint for success + error paths
- Verify HTTP status codes, response JSON structure, and error messages
