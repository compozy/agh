# BUG-002: Missing test coverage for bundle store persistence layer

**Severity:** High
**Priority:** P1
**Component:** `internal/store/globaldb/global_db_bundles.go`
**Related:** TC-INT-001, TC-SEC-002

## Environment

- Code review of test files
- No dedicated test file exists for `global_db_bundles.go`

## Description

The bundle persistence layer (`global_db_bundles.go`, 330 LOC) has zero dedicated unit tests. While the service tests use a memory store mock, the actual SQLite implementation is untested. This means:

1. SQL query correctness is unverified
2. Schema constraints (UNIQUE, CHECK, FK) are untested
3. Timestamp parsing/formatting is untested
4. Error handling paths (RowsAffected=0, sql.ErrNoRows mapping) are untested
5. Transaction behavior in `ReplaceBundleActivationInventory` is untested
6. `CountBundleActivationsForExtension` is untested

### Expected Behavior

Each public method should have table-driven tests covering success and error paths, following the project's bridge store test pattern (`global_db_bridges_test.go`).

### Actual Behavior

No test file `global_db_bundles_test.go` exists.

## Impact

- Schema regressions won't be caught until integration tests
- SQL syntax errors could slip through (e.g., wrong column names)
- FK cascade behavior on deletion is assumed but unverified
- Fails the project's 80% coverage requirement

## Suggested Fix

Create `internal/store/globaldb/global_db_bundles_test.go` following the pattern of `global_db_bridges_test.go`:

- TestOpenGlobalDBCreatesBundleTables
- TestGlobalDBBundleActivationCRUD
- TestGlobalDBBundleActivationInventoryReplace
- TestGlobalDBBundleActivationGuardClauses
- TestGlobalDBCountBundleActivationsForExtension
