AGH Website & Documentation Architecture

Fumadocs on Next.js · Two content collections · Shared design tokens · Vercel static export

Two-Collection Design (ADR-003)

  • • Runtime docs → /runtime/* (~50 pages)
  • • Protocol spec → /protocol/* (~13 pages)
  • • Independent sidebars & templates
  • • Protocol-first landing positioning
  • • Trade-off: cross-links need full paths

packages/ui Boundary

  • • Server-safe: tokens.css, tailwind-preset
  • • Client: Button, Badge, Card ("use client")
  • • NOT shared: domain components (stay in web/)
  • • Consumed by both web/ and packages/site
  • • Explicit export map prevents RSC leaks

CLI Reference Codegen

  • • cmd/agh doc subcommand (Cobra)
  • • GenMarkdownTree → raw .md files
  • • post-process adds frontmatter, .md→.mdx
  • • Written to content/runtime/reference/cli/
  • • Triggered by make cli-docs; overwrites on run

Deployment & Build

  • • Static export (output: 'export') → out/
  • • Vercel at agh.compozy.com + previews
  • • turbo caches out/** per workspace
  • • Makefile: site-dev, site-build, cli-docs
  • • Redirects array from day one

Search & Discovery

  • • Fumadocs built-in Orama (client-side)
  • • Indexes both collections together
  • • Result badges: Runtime | Protocol
  • • No external search service
  • • QMD collections populated post Wave 1

Known Risks

  • • Fumadocs multi-source is less documented
  • • Server/client boundary in packages/ui
  • • Protocol pages drift from source RFCs
  • • Static export blocks interactive API docs
  • • URL changes after external linking
