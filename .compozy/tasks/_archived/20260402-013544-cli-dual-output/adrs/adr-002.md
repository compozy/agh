# ADR-002: Charmbracelet Stack for Human Output Rendering

## Status

Accepted

## Date

2026-03-31

## Context

The human-friendly output mode needs a styling and rendering library for colored tables, status indicators, and structured layouts. The Go ecosystem offers several options: Charmbracelet (lipgloss + log), PTerm, fatih/color + stdlib tabwriter, or building from scratch with ANSI codes.

## Decision

Use the Charmbracelet stack: `charmbracelet/lipgloss` for terminal styling and table rendering, and `charmbracelet/log` as an `slog.Handler` replacement for human-readable log output.

## Alternatives Considered

### Alternative 1: PTerm

- **Description**: Batteries-included terminal UI library with pre-built components
- **Pros**: Ready-to-use spinners, tables, trees, progress bars; fast to prototype
- **Cons**: Larger dependency footprint, less composable, opinionated defaults harder to customize
- **Why rejected**: Too heavy for our needs. We need tables and styled text, not interactive TUI components.

### Alternative 2: fatih/color + stdlib tabwriter

- **Description**: Minimal approach with basic color support and stdlib table formatting
- **Pros**: Tiny dependency footprint, simple API
- **Cons**: No table styling (borders, alignment), no structured layouts, significant manual work for anything beyond basic colors
- **Why rejected**: Too low-level. Would require building our own table renderer and layout system.

### Alternative 3: Custom ANSI rendering

- **Description**: Build output formatting from scratch using raw ANSI escape codes
- **Pros**: Zero dependencies, full control
- **Cons**: Significant development effort, must handle terminal capability detection, color degradation, width calculation
- **Why rejected**: Reinventing the wheel. Lipgloss solves all of these problems.

## Consequences

### Positive

- Lipgloss is the de facto standard for Go CLI styling (used by gh, charm tools)
- `charmbracelet/log` implements `slog.Handler`, so zero changes to existing slog call sites
- Adaptive color support (auto-degrades to 256/16/no color based on terminal)
- Lipgloss table component handles column alignment, borders, and styling
- Active maintenance and large community

### Negative

- Adds two new direct dependencies to go.mod
- Charmbracelet libraries pull in transitive dependencies (lipgloss v1 has moderate dep tree)

### Risks

- Risk: lipgloss API breaking changes in major versions → Mitigation: pin to specific major version
- Risk: charmbracelet/log may not cover all slog handler edge cases → Mitigation: wrap with our observer pattern as we do today

## Implementation Notes

- `charmbracelet/lipgloss` used in `internal/cli/human/` package for all styled rendering
- `charmbracelet/log` used in `internal/logger/logger.go` as alternative handler when format is `human`
- Install via `go get github.com/charmbracelet/lipgloss@latest` and `go get github.com/charmbracelet/log@latest`
- `mattn/go-isatty` is already an indirect dependency (via gin), no new transitive deps for terminal detection if needed later
