# ADR-003: Greenfield Migration Without Backward Compatibility

## Status

Accepted

## Date

2026-03-31

## Context

Migrating roles from TOML to Markdown with YAML frontmatter is a breaking change. Existing role `.toml` files on disk (in `~/.agh/roles/` and `.agh/roles/`) will not be recognized after the migration. Two strategies exist: (1) support both formats with a compatibility layer, or (2) break cleanly and require users to recreate or manually convert their files.

The project is in early development. The number of users with custom roles on disk is negligible.

## Decision

Perform a greenfield migration with no backward compatibility layer. The loader only recognizes `.md` files after the migration. Old `.toml` role files are silently ignored (not loaded). No automatic migration tool is provided.

## Alternatives Considered

### Alternative 1: Dual-Format Support with Deprecation Period

- **Description**: Accept both `.toml` and `.md` role files, with `.toml` emitting a deprecation warning. Remove `.toml` support in a future release.
- **Pros**: Smooth transition for existing users; no data loss.
- **Cons**: Doubles the parsing paths; requires maintaining two codepaths; adds complexity to `parseCatalogFileName`; extends the migration timeline.
- **Why rejected**: The project is pre-stable; the user base with custom roles is negligible. The complexity cost outweighs the migration benefit.

### Alternative 2: Automatic Migration Tool

- **Description**: Provide an `agh migrate` command that converts `.toml` roles to `.md` format.
- **Pros**: One-time effort for users; preserves existing roles.
- **Cons**: Additional code to build, test, and maintain for a one-time operation; still needs to handle edge cases in TOML-to-YAML conversion.
- **Why rejected**: Not justified for the current user base size. Can be added later if demand arises.

## Consequences

### Positive

- Clean, simple implementation with no legacy codepaths.
- No dual-format ambiguity in the loader.
- Reduces code surface area (removes TOML imports from roles/discovery).

### Negative

- Users with existing custom `.toml` roles must manually recreate them.
- Bundled roles installed to `~/.agh/roles/` by previous versions become orphaned `.toml` files.

### Risks

- Users may not notice their roles stopped loading. Mitigation: the CLI `agh role list` will show no roles, prompting investigation.

## Implementation Notes

- `LoadRoles` changes suffix parameters from `.toml`/`.draft.toml` to `.md`/`.draft.md`.
- `bundled.go` `Install` writes `.md` files instead of `.toml`.
- No format detection or fallback logic.
