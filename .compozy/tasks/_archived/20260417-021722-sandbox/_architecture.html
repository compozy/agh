AGH Sandbox — Execution Environment Architecture

Provider abstraction for local + Daytona ACP runtimes · techspec-based view

Core Abstractions (ADR-001)

  • • Provider — Prepare/Sync/Destroy lifecycle
  • • Launcher — returns a Handle with stdin/stdout
  • • ToolHost — fs + terminal + permission policy
  • • Handle — uniform process contract
  • • Providers run in-process, not as extensions
  • • Extension hooks provide extensibility boundary

Daytona Transport (ADR-002)

  • • SSH non-PTY gives clean JSON-RPC stdio pipes
  • • Step 9 is a blocking gate — validate before build
  • • Token-based auth · 60-min expiry · 30s keepalive
  • • Internal transport interface enables WebSocket fallback
  • • One SSH client per sandbox, one session per agent
  • • SDK behind adapter · pinned commit hash

Session + Sync Model (ADR-003)

  • • One sandbox per session · no sharing
  • • Copy-on-start (SyncToRuntime) at session start
  • • Collect-on-stop (SyncFromRuntime) at session stop
  • • Last-write-wins on workspace divergence
  • • Local path stays canonical · runtime path for agent IO
  • • Boot reconciliation cleans orphaned sandboxes

Security & Permissions

  • • Permission approval stays daemon-local
  • • ToolHost enforces policy on runtime paths (no split-brain)
  • • Allowlist env propagation · never leak DAYTONA_API_KEY
  • • environment.prepare hook injects env_overrides
  • • Sync hooks support Deny + exclude_patterns
  • • Network policy best-effort enforced per provider

Config & Workspace Surface

  • • Workspace.EnvironmentRef → Config.Environments key
  • • Cascade: workspace → Defaults.Environment → local
  • • CLI: workspace add/edit --environment <name>
  • • Session list/get/status expose backend + instance_id
  • • SessionEnvironmentMeta persisted in sessions table
  • • No new fields on session create request

Build Order & Risk Gates

  • • 1–3: types, config, workspace resolution
  • • 4–5: extract Launcher + ToolHost from acp (zero behavior change)
  • • 6–8: local provider + session integration + daemon wiring
  • • 9: Daytona SSH non-PTY spike (blocking gate)
  • • 10–11: sshTransport + Daytona provider
  • • 12–14: boot reconciliation, hooks, integration tests
