# ADR-003: AGH_* Environment Variable Prefix Standardization

## Status

Accepted

## Date

2026-04-01

## Context

The AGH codebase currently uses two environment variable prefixes interchangeably: `COLLAB_*` and `AGI_*`. Every point that reads environment variables checks both prefixes with fallback chains (e.g., `COLLAB_SESSION_ID` then `AGI_SESSION_ID`). This duality exists across drivers (`buildEnv()`), the CLI (`agh hook-event`), and will propagate into global plugins.

The project is in alpha and has no external consumers depending on these variable names. This is the ideal time to standardize.

## Decision

Migrate all environment variables to a unified `AGH_*` prefix. Remove all `COLLAB_*` and `AGI_*` references from the codebase.

| Before (dual) | After |
|----------------|-------|
| `COLLAB_SESSION_ID` / `AGI_SESSION_ID` | `AGH_SESSION_ID` |
| `COLLAB_SOCKET` / `AGI_SOCKET` | `AGH_SOCKET` |
| `COLLAB_AGENT` / `AGI_AGENT` | `AGH_AGENT` |
| `COLLAB_AGENT_NAME` / `AGI_AGENT_NAME` | `AGH_AGENT_NAME` |
| `COLLAB_SESSION` / `AGI_SESSION` | `AGH_SESSION_ID` (unified) |

No backward compatibility shims. The project is alpha — clean break.

## Alternatives Considered

### Alternative 1: Keep Dual Prefixes

- **Description**: Continue supporting both `COLLAB_*` and `AGI_*` with fallback chains.
- **Pros**: No migration effort. No risk of breaking anything.
- **Cons**: Every new consumer (plugins, scripts, tests) must check both prefixes. Confusing for contributors. Doubles the surface area for bugs.
- **Why rejected**: The project is alpha with no external consumers. The cost of migration is low now and grows with every new feature.

### Alternative 2: Standardize on AGI_*

- **Description**: Keep `AGI_*` as the prefix, drop `COLLAB_*`.
- **Pros**: Less renaming (AGI_* is already used in more places).
- **Cons**: `AGI` is generic and could conflict with other AI tooling. Does not identify the product.
- **Why rejected**: `AGH_*` is product-specific and unambiguous.

## Consequences

### Positive

- Single, unambiguous prefix across entire codebase
- Plugins reference one set of variable names
- No fallback chains — simpler code
- Product-identifiable prefix (`AGH` = the product name)

### Negative

- Touches many files (drivers, CLI, tests, plugins)
- Any external scripts or configs referencing old vars will break

### Risks

- None significant — project is alpha with no external consumers.
