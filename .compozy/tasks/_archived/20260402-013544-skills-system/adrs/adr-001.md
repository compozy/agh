# ADR-001: Prompt-Only Runtime for Skills

## Status

Accepted

## Date

2026-03-31

## Context

AGH needs a skill execution model. The four reference implementations (Claude Code, Hermes, OpenClaw, OpenFang) offer different approaches: prompt-only (Markdown injected into system prompt), executable runtimes (Python/Node/Shell subprocesses), and WASM sandboxed execution. AGH's kernel is designed as minimal infrastructure — it orchestrates agents but does not execute domain logic.

## Decision

Skills use a **prompt-only runtime**. Each skill is a Markdown document (`SKILL.md`) whose body content is injected into the agent's system prompt. Agents use their existing tools (Read, Write, Bash, etc.) to carry out the instructions. No subprocess execution, no WASM, no sandboxing infrastructure.

## Alternatives Considered

### Alternative 1: Prompt + Executable (Python/Node/Shell)

- **Description**: Support scripts as subprocess tools alongside prompt instructions, following the OpenFang model with 6 runtimes.
- **Pros**: More powerful, skills can perform complex operations autonomously.
- **Cons**: Requires sandboxing, environment isolation, dependency management, and security hardening. Significant complexity increase.
- **Why rejected**: Violates AGH's minimal kernel principle. Agents already have Bash access for running scripts — the kernel does not need to duplicate this capability.

### Alternative 2: Prompt + WASM Only

- **Description**: Support WASM modules for sandboxed extensions alongside prompt instructions.
- **Pros**: Secure by default, language-agnostic.
- **Cons**: WASM ecosystem for this use case is immature. Limited tooling. Premature abstraction for current needs.
- **Why rejected**: No immediate demand. Can be added later without breaking the prompt-only foundation.

## Consequences

### Positive

- Zero execution infrastructure to build or maintain
- Inherently secure — skills cannot execute code directly
- Compatible with all AgentSkills-compliant tools (30+ and growing)
- Simple to author, test, and distribute

### Negative

- Skills cannot perform autonomous computation without agent tool access
- Complex workflows require well-written instructions rather than executable logic

### Risks

- If a skill's instructions are ambiguous, agent behavior may be inconsistent. Mitigation: provide clear authoring guidelines and examples.

## Implementation Notes

The prompt-only model aligns with the AgentSkills specification at https://agentskills.io, which defines skills as Markdown instruction documents. All progressive disclosure tiers (catalog → instructions → resources) operate on text content.
