# ADR-004: Four-Level Loading Hierarchy with .agents/ Compatibility

## Status

Accepted

## Date

2026-03-31

## Context

Skills can be installed at different scopes: bundled with the binary, per-user, per-project, or shared across tools. The AgentSkills ecosystem uses `.agents/skills/` as a cross-client interoperability convention. AGH needs its own namespace (`.agh/skills/`) while remaining compatible with the broader ecosystem.

## Decision

Load skills from **four levels** with strict precedence (highest overrides lowest):

1. `.agh/skills/` — Workspace-level (highest priority)
2. `.agents/skills/` — Cross-client interop (project-level)
3. `~/.agh/skills/` + `~/.agents/skills/` — User-level (global)
4. Bundled (embedded via `go:embed`) — Lowest priority

Skills with the same `name` at a higher level override those at a lower level. A warning is logged when a collision occurs.

## Alternatives Considered

### Alternative 1: Three Levels (No .agents/)

- **Description**: Only bundled → user → workspace. No `.agents/skills/` scanning.
- **Pros**: Simpler. Fewer directories to scan.
- **Cons**: Skills installed by Claude Code, Codex, or other AgentSkills tools are invisible to AGH.
- **Why rejected**: Breaks cross-client interoperability, which is a core goal.

### Alternative 2: Two Levels (User + Workspace Only)

- **Description**: No bundled skills. Users install everything manually.
- **Pros**: Minimal. No embedded assets.
- **Cons**: No out-of-the-box experience. New users get zero skills.
- **Why rejected**: Poor first-run experience.

## Consequences

### Positive

- Skills installed by any AgentSkills-compliant tool are automatically visible to AGH
- Workspace skills can override user/bundled defaults for project-specific needs
- Bundled skills provide a useful default set out-of-the-box

### Negative

- More directories to scan at boot
- Override semantics may confuse users who don't understand the precedence

### Risks

- Scanning large directories could slow boot. Mitigation: max depth of 4 levels, max 300 candidates per root, skip `.git/` and `node_modules/`.

## Implementation Notes

Scanning order during `LoadAll()`: bundled first, then user dirs, then `.agents/`, then `.agh/`. Each subsequent level overrides same-name skills from earlier levels. The `SkillSource` enum tracks provenance.
