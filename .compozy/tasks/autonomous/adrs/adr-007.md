# ADR-007: Minimal Network Evolution for Local Autonomy

## Status

Accepted

## Date

2026-04-25

## Context

The autonomy analysis proposed many network protocol improvements, including multi-home sessions, contract-net verbs, mentions, voting, reactions, escalation, and cross-daemon coordination. The approved scope focuses on local daemon autonomy first while retaining network affordances that help agents coordinate.

## Decision

Keep the first network evolution minimal:

- Keep single-channel sessions for the first autonomy implementation.
- Add or document a minimal channel manifest for discovery metadata.
- Add simple `status` semantics for peer state such as idle, working, blocked, and done.
- Add `handoff` only when needed for auditable transfer of work/context.
- Defer contract-net, multi-home sessions, vote/react/escalate/offer/accept/decline, and complex mention routing.

Any network wire-format bump must be tied to implemented fields/kinds, not done speculatively.

## Alternatives Considered

### Alternative 1: Full network protocol expansion now

- **Description**: Add all proposed verbs and fields in one `agh-network/v1` bump.
- **Pros**: One clean protocol change.
- **Cons**: Overfits unproven coordination patterns and increases implementation scope.
- **Why rejected**: Local autonomy can ship with a smaller network surface.

### Alternative 2: No network changes

- **Description**: Keep network v0 unchanged.
- **Pros**: Lowest protocol risk.
- **Cons**: Leaves agents without first-class status or handoff semantics.
- **Why rejected**: Minimal coordination semantics are useful for autonomous workflows.

## Consequences

### Positive

- Avoids overengineering the network layer.
- Keeps local autonomy independent from cross-daemon swarm design.
- Leaves room for a future protocol bump once usage is real.

### Negative

- Some advanced multi-agent patterns remain out of scope.
- Cross-daemon negotiation remains deferred.

### Risks

- Risk: local autonomy needs richer network semantics sooner than expected.
- Mitigation: use envelope `ext` and task metadata for provisional metadata before formalizing more verbs.

## Implementation Notes

Prefer additive fields and extension conventions until a protocol bump is justified by concrete implementation tasks.
