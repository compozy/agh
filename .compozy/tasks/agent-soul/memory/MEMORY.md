## Shared Learnings
- Later tasks should reuse `soul.Resolve` / `soul.Parse` and the exported result/read-model/projection structures instead of reparsing `SOUL.md` in session, task, API, CLI, extension, or authoring code.
- Invalid existing `SOUL.md` content returns typed diagnostics with `Present=true`, `Valid=false`, `Active=false`, and `ReadModel.Valid=false`; missing optional files return `Present=false` without diagnostics.
- Task 02 stores query authority in typed global DB columns (`agent_soul_snapshots`, `agent_soul_revisions`, and `sessions.soul_*`) and stores full read-model/compact/config provenance details in the snapshot `profile_json` envelope. Later tasks should use the store methods instead of parsing raw `profile_json` or re-resolving files in session/task claim code.
- Global DB migration v12 is `add_agent_soul_snapshots` from commit `538955da`; migration v13 is `add_agent_heartbeat_storage` from commit `8096a2fc`.
- Later route/contract tasks should expose Soul data from session metadata, stored snapshots, and `internal/situation` compact context rather than rereading `SOUL.md`.
- Later Heartbeat tasks should reuse `heartbeat.Resolve`, `heartbeat.Parse`, `heartbeat.ConfigProvenanceFor`, and `ResolvedPolicy` instead of reparsing `HEARTBEAT.md`; invalid present Heartbeat policy fails closed for wake policy only, while missing optional files remain valid/inactive.
- Heartbeat snapshots store normalized policy body plus frontmatter JSON, while Heartbeat revisions preserve the authored source body. Rollback by digest must reconstruct source from snapshot frontmatter/body; rollback by revision can use the revision body directly.
- Task 10 split the compact context Soul projection into `AgentSoulSectionPayload`; use `AgentSoulPayload` only for the dedicated full read/inspect/authoring surfaces.
- Mutating Soul and Heartbeat transport DTOs now use body-level `expected_digest`; keep HTTP `If-Match` out of future HTTP/UDS/CLI/Host API parity work.
- Task 11 enforces the Heartbeat `If-Match` exclusion at the shared handler boundary with a deterministic 400 before service mutation.
- Workspace-resolved config must preserve the full `[agents]` block; dropping `Agents` during workspace config cloning zeroes Soul/Heartbeat defaults and breaks managed authoring/status validation.
- `make codegen` writes `openapi/agh.json` with 2-space indent and one-element-per-line arrays, but the committed file historically uses 4-space indent with compact short arrays. Running codegen blindly produces a ~130k-line whitespace-only diff. Future contract edits should patch the targeted operation spans surgically (canonical-JSON `codegen-check` ignores indent/key-order differences) until a dedicated reformat task realigns the writer with the committed style.
- `make cli-docs` (cobra → docpost) emits unaligned table columns; run `bunx oxfmt packages/site/content/runtime/cli-reference` after every regeneration to keep diffs scoped to actual command changes. `oxfmt` also formats `.mdx` files but lint-staged only auto-runs on `.md`, so explicitly format edited Fumadocs pages before committing.
