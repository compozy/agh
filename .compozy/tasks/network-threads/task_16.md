# Task 16: Site, Runtime Docs, Examples, and CLI Reference Co-Ship

## Overview

Update public and runtime documentation after the implementation surfaces exist. This task rewrites site docs, protocol docs, examples, API reference, CLI reference, tool docs, extension docs, SDK docs, and landing visuals/copy around `surface`, `thread_id`, `direct_id`, and `work_id`.

<critical>
- ALWAYS READ `_techspec.md`, `_design.md`, all ADRs, `packages/site/CLAUDE.md`, `COPY.md`, `DESIGN.md`, `docs/_memory/glossary.md`, and final CLI/native/Host API outputs before editing.
- ACTIVATE `documentation-writer`, `copywriting`, `crafting-effective-readmes`, `agh-design`, `design-taste-frontend`, and `minimalist-ui` for public docs/visual surfaces.
- REFERENCE implemented behavior, generated API docs, and generated CLI docs; runtime truth beats aspirational copy.
- FOCUS ON truthful operator and agent guidance.
- TESTS REQUIRED for docs build, source generation, generated references, snippets, links, and legacy-term scans.
- NO WORKAROUNDS: do not hand-edit generated API/CLI reference output when generators exist.
</critical>

<requirements>
- MUST rewrite runtime network docs around channels, public threads, direct rooms, work IDs, delivery/safety, and task ingress.
- MUST rewrite protocol docs around envelope, message kinds, examples, NATS terminology, conformance, and interactions/work.
- MUST remove public active references to `interaction_id` and `kind:"direct"`.
- MUST distinguish NATS transport "direct" terminology from `surface:"direct"` direct rooms where transport docs require it.
- MUST regenerate API reference from OpenAPI and CLI reference from Cobra/generator paths where applicable.
- MUST document native tools, hosted tool schemas, extension Host API, SDK exports, bridge mapping limits, and hook observation semantics.
- MUST document config lifecycle as no new conversation config keys and no unsupported settings controls.
- MUST keep direct-room language to restricted visibility, not cryptographic privacy.
</requirements>
