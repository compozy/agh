# AGH Network Threads and Direct Rooms - Task List

**GREENFIELD (alpha):** do not preserve legacy behavior for old local state. These tasks implement the approved AGH Network threads TechSpec by hard-cutting the protocol, runtime, store, API, CLI, native tools, extension surfaces, web UI, docs, and verification harness from flat channel/peer timelines to explicit public threads, direct rooms, and work metadata.

The normative design inputs are `_techspec.md`, `_design.md` (web/UI), and ADR-001 through ADR-003. This task list intentionally contains implementation and support tasks only. QA report and QA execution tasks are generated by `.compozy/extensions/cy-qa-workflow/extension.toml` through required workflow hooks, so they are not authored manually here.

## MVP Boundary

MVP includes channel-scoped public threads, channel-scoped two-party direct rooms, `surface:"thread"|"direct"`, hard-cut `interaction_id -> work_id`, deletion of `kind:"direct"`, durable conversation containers, store-backed work lifecycle metadata, HTTP/UDS/CLI/native-tool/extension/web manageability, agent prompt and bundled skill semantics, site/runtime docs, and E2E fixture alignment. MVP excludes private threads, group direct rooms, unread sync, notification preferences, retention controls, new `config.toml` keys, and any compatibility shim for `interaction_id` or `kind:"direct"`.

## Web/UI Source

Web work is split across **task_13** (shell + routes + IA + query isolation), **task_14** (message row + timeline + thread overlay + author group collapse), and **task_15** (composer + work surfacing + empty/error states + realtime polling). The normative UI source for all three is `_design.md` in this directory. Where `_design.md` and `_techspec.md` overlap, `_techspec.md` wins for protocol/data semantics; `_design.md` wins for layout, interaction, visual treatment, and information architecture.
