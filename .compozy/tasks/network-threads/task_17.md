# Task 17: E2E Harness and Fixture Alignment

## Overview

Align runtime and browser verification fixtures with the final conversation model before the QA workflow extension generates release-grade QA tasks. This task updates acpmock, daemon collaboration harnesses, browser artifacts, selectors, and E2E seeds so downstream QA validates thread/direct/work behavior instead of legacy peer-room timelines.

<critical>
- ALWAYS READ `_techspec.md`, `_design.md`, all ADRs, `internal/CLAUDE.md`, `web/CLAUDE.md`, and final implementation task outputs before editing.
- ACTIVATE `agh-test-conventions`, `testing-anti-patterns`, `real-scenario-qa` for scenario shape, `qa-report` for coverage thinking, and `agh-worktree-isolation` if running isolated E2E locally.
- REFERENCE TECHSPEC for required QA shape, but do not manually create QA report/execution task files because `cy-qa-workflow` generates them.
- FOCUS ON harness and fixture readiness, not replacing per-task unit/integration tests.
- TESTS REQUIRED for acpmock wrapper assertions, daemon collaboration flows, browser artifacts, and web E2E fixture fields.
- NO WORKAROUNDS: do not preserve old `network_selected_peer` artifacts as active fields.
</critical>

<requirements>
- MUST update acpmock matchers and fixtures to assert `surface`, container ID, `work_id`, `reply_to`, `trace_id`, `causation_id`, and trust metadata.
- MUST update daemon network collaboration integration assertions for public threads, restricted direct rooms, handoff, summarize-back, and direct resolve race.
- MUST update runtime harness helpers and scenario seeds around new message payloads.
- MUST update browser artifact fields from `network_selected_peer` to `network_selected_thread` and `network_selected_direct`.
- MUST update Playwright selectors and E2E fixtures for thread list, thread detail, direct list, direct detail, direct resolve, and summarize-back workflows.
- MUST preserve raw-token redaction assertions across prompt, tools, logs, audit, and browser artifacts.
- MUST leave QA report/execution task creation to `.compozy/extensions/cy-qa-workflow/extension.toml`.
</requirements>
