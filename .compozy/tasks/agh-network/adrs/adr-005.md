# ADR-005: Runtime-Created Spaces with Explicit Session Opt-In

## Status

Accepted

## Date

2026-04-10

## Context

AGH Network uses "spaces" as logical communication namespaces. Sessions join spaces to participate in agent-to-agent communication. The design questions are: (1) where do space definitions live (config vs database), (2) how do sessions join spaces (auto-join vs opt-in), and (3) what runtime state needs persistence.

Three advisor perspectives were evaluated:
- Architect: spaces should stay lightweight and ephemeral, with only network defaults in config
- Pragmatic Engineer: minimum config (`enabled`, `default_space`, port), no persisted space catalog in v0
- Product Mind: auto-join a default "local" space for zero-config experience

## Decision

Spaces are runtime-created on first reference. The `[network]` config section carries defaults and transport settings (`enabled`, `default_space`, `port`, heartbeat and replay limits), but it does not define a catalog of named spaces. Sessions opt-in explicitly via `--space` on `agh session create`. Sessions without `--space` remain isolated. Space membership and peer presence are in-memory and reconstructable from active sessions plus periodic `greet` traffic. Audit history is persisted separately; space definitions are not.

## Alternatives Considered

### Alternative 1: Auto-Join Default Space

- **Description**: Every session joins a "local" space automatically when network is enabled
- **Pros**: Zero-config networking, immediate agent visibility
- **Cons**: Breaking change to session mental model (sessions are currently isolated), noise for sessions that don't need networking
- **Why rejected**: Violates zero-surprise principle. Network participation should be intentional.

### Alternative 2: Database-Persisted Spaces

- **Description**: Spaces stored in SQLite globaldb with CRUD API, membership history
- **Pros**: Survives daemon restart, queryable history, supports dynamic space management
- **Cons**: Premature schema design for an entity whose requirements are unclear, adds migration burden, over-engineering for v0
- **Why rejected**: Spaces are reconstructable from active sessions + greet traffic. Persisting a catalog adds complexity without clear v0 benefit.

### Alternative 3: Agent-Level Space Declaration

- **Description**: AGENT.md frontmatter declares which spaces the agent joins, sessions auto-join
- **Pros**: Network config lives with agent definition, no per-session flags needed
- **Cons**: Mixes network topology with agent identity, all sessions of an agent join the same spaces
- **Why rejected**: Per-session control is more flexible. Agent-level config can be added later as a default that --space overrides.

## Consequences

### Positive

- Zero behavior change for existing sessions
- Config still follows the established TOML overlay pattern for defaults and transport settings
- No space catalog DB schema to maintain or migrate
- Explicit opt-in makes network participation intentional and observable
- Runtime space state is reconstructable — no persisted catalog = no stale space definition bugs

### Negative

- Requires explicit --space flag (one extra CLI argument)
- Spaces lost on daemon restart until sessions re-join and peers re-greet — acceptable for v0
- No persisted space catalog; operators inspect active spaces at runtime instead

### Risks

- User forgets --space flag and wonders why agents can't communicate: mitigated by clear error messages and skill instructions

## Implementation Notes

- Config: `[network] enabled = false, default_space = "default"` for defaults only, not a list of allowed spaces
- Session creation: `agh session create --agent reviewer --space builders`
- Extend `CreateOpts` and `CreateSessionRequest` with optional `Space string` field
- Network Manager tracks local membership and an optional remote peer cache keyed by `peer_id`
- On daemon restart: active sessions with persisted `Space` metadata can be re-joined during daemon reconciliation; spaces themselves are not persisted separately
- Future: add AGENT.md `network.spaces` as a defaulting hint, with `--space` as override, if needed
