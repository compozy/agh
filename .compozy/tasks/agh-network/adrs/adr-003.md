# ADR-003: CLI + Bundled Skill for Agent Network Communication

## Status

Accepted

## Date

2026-04-10

## Context

Agents running as ACP subprocesses need a mechanism to send and receive AGH Network messages. The ACP protocol has no inter-agent communication methods — it is purely a 1:1 editor-to-agent protocol. We need a bridge between the agent's capabilities and the network.

Research across five reference projects (Claude Code, Hermes, GoClaw, OpenFang, OpenClaw) shows the universal pattern: agents use tools to send messages, and the framework injects incoming messages as new conversation turns.

Four options were evaluated: (1) MCP server built into the daemon, (2) ACP extension methods, (3) CLI commands via terminal tools, (4) CLI + bundled skill.

## Decision

Use CLI commands for the outbound path (agent sends messages) combined with a bundled skill for contextual instructions. The agent calls `agh network send`, `agh network peers`, etc. via its terminal tool access. A bundled skill `agh-network` provides a SKILL.md with usage instructions, message format examples, and best practices that are appended to the session system prompt whenever a session starts with network participation, including resume flows that restore `Space` metadata.

For the inbound path (network delivers to agent), the daemon auto-prompts the session via `session/prompt` when a network message arrives, with queue-and-deliver-after-turn semantics when the agent is busy.

## Alternatives Considered

### Alternative 1: MCP Server Built-in

- **Description**: Daemon exposes an internal MCP server with typed tools (network_send, network_peers)
- **Pros**: Structured input/output, JSON-native, no subprocess overhead per message
- **Cons**: Requires implementing MCP server in the daemon (new code path), agents need MCP client support
- **Why rejected**: Adds implementation complexity when CLI already provides the same functionality through existing infrastructure (terminal tools, UDS)

### Alternative 2: ACP Extension Methods

- **Description**: Use ACP's ExtMethodRequest/ExtNotification for custom _network/* methods
- **Pros**: Tightest integration with ACP protocol
- **Cons**: Requires agent subprocess to implement extension methods — third-party agents (Claude Code, Codex) would need changes
- **Why rejected**: Depends on upstream agent implementations we don't control

### Alternative 3: SDK Helper in System Prompt

- **Description**: Inject structured output format instructions in system prompt, daemon parses agent output for message patterns
- **Pros**: Zero infrastructure dependency
- **Cons**: Fragile parsing, unreliable formatting from LLMs, no structured error handling
- **Why rejected**: Unreliable — LLM output cannot be trusted for protocol correctness

## Consequences

### Positive

- Reuses existing infrastructure: CLI, UDS transport, terminal tools, skills system
- CLI commands useful for both humans and agents
- Skill provides contextual instructions without hardcoding in system prompts
- Zero new protocol surface — agents use tools they already have (terminal access)
- Can add MCP tools to the skill later without breaking changes

### Negative

- CLI invocation has subprocess overhead per command (~50ms per call)
- Output is text-based, requires JSON parsing from CLI output
- Agent must have terminal tool access enabled

### Risks

- Agent may not reliably use CLI commands: mitigated by skill instructions with explicit examples
- CLI output format changes could break agent parsing: mitigated by using `--output json` flag for structured output

## Implementation Notes

- Bundled skill at `internal/skills/bundled/skills/agh-network/SKILL.md`
- CLI commands under `agh network` namespace: `send`, `peers`, `spaces`, `status`, `inbox`
- CLI communicates with daemon via UDS (existing pattern from all other CLI commands)
- Inbound: Network Manager delivers through a `PromptNetwork` adapter that calls `session.PromptWithOpts(..., TurnSourceNetwork)`
- Queue: messages buffered when session has active turn, delivered FIFO after turn completes
- Skill content appended after prompt assembly and before ACP startup when a session starts with `Space` set, including restored resume state

## References

- Claude Code: SendMessageTool pattern (file-based mailbox + tool call)
- Hermes: send_message tool + inject_message for inbound
- GoClaw: chat.inject RPC method for inbound delivery
- OpenFang: KernelHandle.send_to_agent() + REST API
