# ADR-010: Fact Extraction Location — Hybrid Per-Turn Hook + Optional Compaction Flush

## Status

Accepted

## Date

2026-05-04

## Context

ADR-009 placed the write controller as the single decision point for ADD/UPDATE/DELETE/NOOP/REJECT, with explicit "extraction belongs upstream of `Decide()`." This ADR resolves the remaining question: **where does fact extraction (turn → memory candidates) actually run?**

`analysis_extraction-location.md` verified four production patterns in source code:

- **Claude Code** — `on_post_response` hook fires a forked sub-agent at Stop event with sandboxed tool whitelist (`extractMemories.ts:415-427, 171-222`). Per-turn cadence with feature-flagged throttle. Sub-agents skipped (`extractMemories.ts:531-534`). Async fire-and-forget; drained on shutdown. **Single-slot pending uses silent overwrite** (`extractMemories.ts:553-563`) — the exact pattern Codex peer review §3.7 flagged as unsafe for AGH.
- **OpenClaw** — pre-compaction silent agent turn (`agent-runner-memory.ts:847-872`). Triggered by AND-gated thresholds (token + transcript byte). **Synchronous — blocks next turn** (`agent-runner-memory.ts:799,830-884`). Sets `phase: memory_flushing` visibly.
- **Hermes** — no autonomous extractor. Main agent calls `memory(action, ...)` tool inline. Doesn't scale past hundreds of entries (Hermes' own analysis admits).
- **Codex** — deferred two-phase at next-session boot (`memories/write/src/start.rs:22-75`). Phase 1 per-rollout extraction with `gpt-5.4-mini` (concurrency 8); Phase 2 sandboxed consolidation agent. Skips sub-agents (`source.is_non_root_agent()`). Pays cold-start cost on resume; recent learnings unavailable in same session.

AGH constraints (load-bearing):

- **SD-010 detached lifetime** — long-running extractor calls MUST use `context.WithoutCancel(parent)` with re-attached deadline.
- **Subagents read-only by default** — extractor tool whitelist must enforce.
- **Hooks dispatch at call site** — extraction must dispatch from the package owning assistant-message persistence (`internal/session/manager_lifecycle.go` or `internal/transcript`), never tail event tables.
- **Bounded queue with observable dropped windows** (Codex peer review §3.7) — silent single-slot overwrite is forbidden.
- **Hot-path latency** — extraction must not block user-facing turn response.
- **Agent-manageable** — every extractor operation visible via CLI/HTTP/UDS with structured outputs.

## Decision

Adopt **D2 hybrid** — per-turn `on_post_response` forked extractor (Mode A, default) + optional pre-compaction silent flush (Mode B, Slice 2).

### Mode A — `on_post_response` forked extractor (Slice 1, default)

- **Package**: new `internal/memory/extractor`. Exposes `Extractor` interface and `Forked` concrete implementation.
- **Hook event**: `hook.session.message_persisted` added to `internal/hooks` typed taxonomy. Dispatched from `internal/session/manager_lifecycle.go` at the call site finalizing assistant-message persistence.
- **Subagent execution**: forked from the main session via the existing ACP subagent path. Inherits parent prompt cache when the ACP driver supports it (Anthropic-class drivers); falls back to a configurable lightweight model otherwise (open question Q1, deferred to TechSpec).
- **Tool sandbox** (mirrors Claude Code `createAutoMemCanUseTool`):
  - `read`, `grep`, `glob` — unrestricted
  - `bash` — only when `tool.isReadOnly(parsed)` returns true
  - `edit`, `write` — only when path passes `isAGHInboxPath()` (rooted at `$AGH_HOME/memory/_inbox/`)
  - REPL — allowed (primitive tools route through it)
  - Everything else — denied; emits `memory.extractor.tool_denied` event with tool name + reason
- **Throttle**: `extractor.throttle_turns = 1` (default — every eligible turn). Configurable per-agent via `agent.memory.extractor.throttle_turns`. Trailing runs from queued context bypass throttle.
- **Sub-agent rule**: skip extraction when `actor_kind == "agent_subagent"` OR `parent_session_id != ""`. Codex/Claude Code shape. The hook still dispatches for telemetry, but the extractor consumer no-ops.
- **Async (SD-010)**:
  - Spawn detached: `ctx := context.WithoutCancel(parent); ctx, cancel := context.WithDeadline(ctx, time.Now().Add(deadline))`
  - Default `extractor.deadline = 60s` (configurable)
  - Manager-owned `WaitGroup` tracks in-flight extractions
  - `Extractor.Drain(ctx, drain_timeout)` called from daemon shutdown (default 60s)
  - Re-attach deadline because `context.WithoutCancel` does not preserve deadlines
- **Failure mode** — best-effort:
  - On error: log + emit canonical event `memory.extractor.failed` with correlation keys (`session_id`, `root_session_id`, `agent_id`, `attempt`, `error_class`)
  - DLQ at `$AGH_HOME/memory/_system/extractor/failures/<utc>-<session_id>.json` containing turn payload + error
  - Replayable via `agh memory extractor replay --session <id> [--from-dlq]`
  - Cursor `last_extracted_message_seq` does **not** advance — next turn re-processes the same range
- **Single-slot pending policy** (Codex peer review §3.7 fix):
  - Bounded `chan extractRequest` capacity 1 per session (one in-flight + one queued)
  - On overflow: queued slot is **merged** with the new request (concat message-seq ranges) — never silently overwritten
  - Each merge emits `memory.extractor.coalesced` with `dropped_turn_count`
  - Hard ceiling: `coalesce_max = 16`. Beyond that, drop the **oldest** queued request (retain newest), emit `memory.extractor.dropped` — visible, not silent
- **Output destination**:
  - Candidates written as JSONL to `$AGH_HOME/memory/_inbox/<session_id>/<utc>-<message_seq>.jsonl`
  - One line per `Candidate` (see ADR-009 schema)
  - A single daemon-owned `InboxConsumer` reads from `_inbox`, runs prefilter + LLM tiebreaker through `Controller.Decide()`, commits or rejects, then deletes the processed file on success
  - Producers never read or delete inbox files; decode/apply failures move the file into `_system/extractor/failures/` so replay tooling sees the exact failed payload
- **Mutual exclusion with explicit memory-tool calls**: if main agent invoked `agh__memory_propose` (ADR-009) in the same turn, extractor no-ops for that turn (`hasMemoryWritesSince` analog). Prevents double-write race.

### Mode B — Pre-compaction silent flush (Slice 2)

- **Trigger**: AND-gated:
  - `tokenCount >= contextWindow - reserveTokensFloor - softThresholdTokens` (defaults `reserveTokensFloor = 8000`, `softThresholdTokens = 4000`)
  - OR `transcriptByteSize >= forceFlushTranscriptBytes` (default 8 MiB)
- **Dedup**: only one flush per `compactionCount` value (deterministic, mirrors OpenClaw `memory-flush.ts:118-124`)
- **Scope**: only ships when the compactor itself ships in Slice 2 — depends on Slice 2 deliverable
- **Async**: Mode B is **also detached** for AGH (departure from OpenClaw's sync model). The flush turn runs under `context.WithoutCancel` with its own deadline; the next user turn does not block on it. The compactor itself blocks (it must, since it modifies the active context); Mode B's job is to write durable memory before compaction discards content, not to gate compaction.
- **Output**: same `_inbox/` destination as Mode A. The controller is single, the inbox is single.

### Cascade alignment

The two modes map onto AGH's existing **Time → Sessions → Lock** consolidation cascade:

- A = **Time gate** — every-N-turns cadence
- B = **Sessions/size gate** — compaction-watermark trigger
- Both feed one bounded `_inbox` queue read by `Controller.Decide`

The pre-compaction flush in Mode B does not redo Mode A's work — it consolidates the recent transcript window into denser candidates that Mode A's per-turn cadence may have missed.

### Go interface (sketch — TechSpec finalizes)

```go
// internal/memory/extractor/extractor.go
package extractor

type Trigger string
const (
    TriggerPostMessage     Trigger = "post_message"
    TriggerCompactionFlush Trigger = "compaction_flush"
)

type TurnRecord struct {
    SessionID         string
    RootSessionID     string
    ParentSessionID   string // empty for root sessions
    AgentID           string
    ActorKind         string // "agent_root" | "agent_subagent" | ...
    WorkspaceID       string
    SinceMessageSeq   int64  // exclusive
    UntilMessageSeq   int64  // inclusive
    Snapshot          TranscriptSnapshot
    Trigger           Trigger
}

type Extractor interface {
    // Extract runs in a detached context. Returns candidates evaluated by
    // the controller. Empty + nil err is a valid no-op.
    Extract(ctx context.Context, turn TurnRecord) ([]Candidate, error)

    // Drain blocks until in-flight extractions finish or the context expires.
    Drain(ctx context.Context) error
}
```

### Configuration keys

```toml
[memory.extractor]
enabled            = true
mode               = "post_message"          # post_message | compaction_flush | hybrid
throttle_turns     = 1
deadline           = "60s"
sandbox_inbox_only = true                    # forced true; key reserved for future loosening
inbox_path         = "$AGH_HOME/memory/_inbox"
dlq_path           = "$AGH_HOME/memory/_system/extractor/failures"
model              = ""                      # empty = forked from main; otherwise e.g. "anthropic/claude-haiku-4"

[memory.extractor.queue]
capacity           = 1                       # in-flight + queued; coalesce on overflow
coalesce_max       = 16                      # hard drop threshold

[memory.extractor.compaction_flush]          # Mode B (Slice 2)
soft_threshold_tokens         = 4000
reserve_tokens_floor          = 8000
force_flush_transcript_bytes  = "8MiB"
```

All keys hot-reloadable.

### Agent-manageable surface (mandatory per SD-011)

- `agh memory extractor status [--session <id>] [-o json]` — current queue depth, last extraction, last failure
- `agh memory extractor list-pending [-o jsonl]` — entries currently in the bounded queue
- `agh memory extractor replay --session <id> [--from-dlq]` — re-extract a session's failed turns
- `agh memory extractor drain [--timeout 60s]` — operator-triggered drain
- `agh memory extractor disable [--session <id>]` — operator override
- HTTP/UDS parity for each (per `internal/CLAUDE.md` "Agent-operable by default")

## Alternatives Considered

### Alternative 1: Mode A only (Claude Code style, no compaction flush)

- **Description**: Just `on_post_response` per-turn forked extractor. No pre-compaction flush.
- **Pros**: Single code path. Simpler.
- **Cons**: Misses the "save before compaction discards" moment. When a session has a long, dense burst of context that compaction is about to truncate, Mode A's per-turn cadence may not have produced strong candidates yet.
- **Why rejected**: Mode B is cheap to add (it's an explicit subset of Mode A's machinery, called from the daemon's compaction trigger). Adding it later would be a breaking change to the extractor contract.

### Alternative 2: Mode B only (OpenClaw style)

- **Description**: Only run extraction when compaction fires.
- **Pros**: Single trigger, deterministic dedup per compaction cycle.
- **Cons**:
  - Sync-blocks next turn in OpenClaw's reference design — violates AGH hot-path-latency rule. AGH's adaptation would need to make B async.
  - Depends on a working compactor (Slice 2 deliverable); not available for the slice that ships extractors.
  - Misses turns in **short sessions** that never hit compaction threshold (the most common operator session: short clarification chats).
- **Why rejected**: Operator UX requires per-turn extraction; short sessions must produce memory too.

### Alternative 3: Mode D (Codex deferred two-phase)

- **Description**: Extraction at next-session boot, never during the session being recorded.
- **Pros**: Zero in-session cost. Durable state-DB lease pattern. Per-rollout claims serialize concurrent extractors.
- **Cons**:
  - **Cold-start cost on resume**: every root session boot pays an extraction backlog. For AGH's local-first single-binary runtime, this is felt by the operator as high CPU on a fresh boot.
  - 30-day window means a freshly-installed AGH has zero memory until the user has used it for at least one full session and *then* booted a second session.
  - Recent learnings **not available in the session that produced them** — only in the next session. Hurts UX for "AGH should remember what we decided 5 minutes ago."
- **Why rejected**: Operator experience prioritizes same-session recall. Codex's design is correct for their operator model (CLI invocations are short, restart-driven); AGH's daemon-resident model with long-lived sessions has different ergonomics.

### Alternative 4: Mode C (extraction inside `Controller.Decide`)

- **Description**: Caller passes raw turn content to `Decide()`; `Decide()` extracts + decides in one call.
- **Pros**: Simpler caller surface (one method).
- **Cons**: Each `Decide()` pays 1-2 LLM calls; breaks ADR-009's sync sub-300ms latency budget; inverts separation of concerns.
- **Why rejected**: `analysis_write-controller.md` already established that extraction belongs upstream of the controller.

## Consequences

### Positive

- Per-turn cadence keeps candidate stream fresh; no 30-day backlog at boot.
- Zero user-visible latency (fire-and-forget after assistant-message persistence).
- Aligns with existing typed hook taxonomy (`internal/hooks` already dispatches at call site).
- Forked sub-agent reuses parent prompt cache when ACP driver supports it.
- DLQ + replay surface mean failures are recoverable, not silent data loss.
- Bounded queue with merge-on-overflow + canonical events fixes Codex peer review §3.7 concern.
- Mode B graduates with the compactor; no architectural rework needed.
- Cascade alignment (Time/Sessions/Lock) makes the design legible to existing AGH developers.

### Negative

- Two complementary cadences mean two trigger paths to maintain. Mitigated: Mode B is an explicit subset of Mode A's machinery.
- Per-turn extractor cost adds up across many concurrent sessions (e.g., 10 active sessions × 3 turns/min × forked Sonnet ≈ 30 model calls/min). Mitigated: configurable `throttle_turns`, configurable `model` for cheaper extraction, mutual exclusion when main agent already wrote.
- Tuning surface is larger (per-turn N, plus token threshold, plus byte threshold for Mode B). Mitigated: defaults match production-validated competitor values; eval harness validates.

### Risks

- Forked sub-agent without prompt-cache reuse blows up token cost on non-Anthropic ACP drivers. Mitigation: `extractor.model` config key allows operator to set a cheaper dedicated model; falls back to forked-from-main when empty.
- Mode A extractor falls behind during a session burst → queue overflow → coalesce/drop events. Mitigation: queue caps + visibility + `agh memory extractor status` for operators to monitor.
- Sub-agent extraction skip means sub-agent-only sessions never produce memory. Mitigation: parent root-session extraction runs after sub-agent returns and sees the full transcript projection; the root captures the lessons.
- Race between explicit `agh__memory_propose` and extractor: if both fire for the same turn, the extractor's no-op gate triggers via `hasMemoryWritesSince`. If the propose call lands AFTER extractor has already started, the controller's hash dedup (ADR-009 NOOP path) catches it. Mitigation: tested in slice 1 QA proofs.

## Implementation Notes

- New package: `internal/memory/extractor` — owns the `Extractor` interface, `Forked` implementation, queue, sandbox tool whitelist, DLQ writer.
- Hook event registration: `hook.session.message_persisted` added to `internal/hooks/dispatch_events.go`.
- Dispatched from `internal/session/manager_lifecycle.go` at the assistant-message persistence transition (call-site rule).
- Sub-agent invocation: reuses existing ACP subagent path with `internal/acp` driver; configures via `extractor.model` if non-empty.
- DLQ format: JSON with `session_id`, `attempt`, `error`, `turn_payload` (compressed transcript snapshot), `enqueued_at`, `coalesced_with` (array of merged turn ranges).
- `agh memory extractor *` CLI verbs ship in slice 1 alongside the extractor itself.
- Schema migration: no new tables (state lives in process memory + filesystem JSONL). State DB is unchanged for slice 1; slice 2 may add lease tables for Mode B if needed.
- Eval harness: synthetic transcripts in `internal/memory/extractor/testdata/` validate (a) per-turn extraction correctness, (b) coalesce-merge semantics, (c) DLQ replay, (d) sub-agent skip, (e) mutual exclusion with explicit memory-tool calls.

### Sub-questions deferred to TechSpec

1. **Forked sub-agent vs dedicated lightweight model** — default policy across ACP drivers. Currently `model = ""` falls back to forked-from-main; benchmark whether haiku-class dedicated wins on ACP drivers without cache reads.
2. **Coalesce-merge semantics** — exact merge rule for two `TurnRecord` ranges. Concat `[since, until]`? Union with gap detection? TechSpec must commit.
3. **Sub-agent inclusion in Mode B** — extract sub-agent transcripts inline in root flush vs per-sub-agent flush.
4. **Per-rollout vs per-turn claim** for Mode B durability.
5. **Cross-session content-hash dedup** of identical candidates before model call — cheap pre-controller dedup vs leaving dedup to the controller.
