# ADR-006: Session Ledger Hybrid (events.db Live + ledger.jsonl Forensic)

## Status

Accepted

## Date

2026-05-04

## Context

AGH today journals session events into `events.db` (per `analysis_agh-current.md`) — append-only event store, transactional, fast. There is no on-disk JSONL ledger; operators who want to inspect a session's history query events.db via API or wait for it to surface in observability tooling.

The user ("I keep a ledger in my projects") wants a **persistent forensic record** per session: something they can `cat`, `grep`, `tar`, replay, or browse months later. The question is *where* and *how* without complicating the live write path.

Source-verified competitor patterns (`analysis_session-ledger-retention.md`):

- **Codex** — `~/.codex/sessions/YYYY/MM/DD/rollout-<ts>-<uuid>.jsonl` is **filesystem-authoritative** (`codex-rs/rollout/src/recorder.rs:1363-1393`). SQLite is the index. No time-based retention on rollout files. Dating subdirectory (YYYY/MM/DD) provides natural archival.
- **Hermes** — pure SQLite (`~/.hermes/state.db`), `prune_sessions(older_than_days=90)` at `hermes_state.py:2074-2127`; auto-prune idempotent every 24h via `state_meta.last_auto_prune` flag.
- **OpenClaw** — JSONL at `~/.openclaw/agents/<agentId>/sessions/<sid>.jsonl` + `sessions.json` index. Three caps stack (`store-maintenance.ts:10-15`): 30 days + 500 entries + configurable disk budget.
- **Claude Code** — per-session JSONL at `~/.claude/projects/<sanitized-cwd>/<sessionId>.jsonl`. **No retention.**
- **OpenFang** — `<workspace>/sessions/<session-id>.jsonl` mirror. **No GC.**

Three of five competitors put session ledgers in **$HOME-based global location**; only OpenFang puts in workspace.

The user's intuition (`~/.agh/sessions/<workspace_id>/<session_id>/ledger.jsonl`) aligns with the majority pattern.

## Decision

Adopt **hybrid live + forensic ledger**:

- **Live state**: `events.db` (per-workspace, ADR-003) is **authoritative during the session.** All event writes happen in transactional SQLite. Fast, durable, queryable.

- **Forensic snapshot**: on `OnSessionEnd`, the daemon **materializes** a `ledger.jsonl` by exporting all events for that session into a single append-only JSONL file at:

  ```
  ~/.agh/sessions/<workspace_id>/<session_id>/
  ├── meta.json         # already exists today; expand with workspace_id, agent_name, ended_at
  └── ledger.jsonl      # NEW — materialized at session end
  ```

  For sessions that are not workspace-bound (e.g., `agh chat` from an arbitrary directory), use `_unbound` as the workspace_id partition:

  ```
  ~/.agh/sessions/_unbound/<session_id>/ledger.jsonl
  ```

- **`ledger.jsonl` format**: one JSON object per line, sorted by event sequence. Line 1 is `ledger_meta {version, session_id, workspace_id, spawn_parent_id?, started_at, ended_at}` so the file is self-describing. Subsequent lines are ordered session-event records (transcript + memory + lifecycle + redaction metadata) rather than a lossy projection of `memory_events` alone.

- **Lifecycle**:
  - Session starts: events go to events.db. No JSONL writes.
  - Session ends: dispatch `OnSessionEnd` hook → materializer reads all events for `session_id`, writes JSONL atomically (`mkstemp + fsync + rename` per Hermes `_write_file`).
  - JSONL is **read-only** after materialization. Re-running materialization is idempotent (checksum on existing file before overwrite).
  - `agh sessions list/show/replay <session_id>` reads from JSONL when present (forensic surface); falls back to events.db when JSONL hasn't been materialized yet (live session).
  - Session ledgers are **forensic only**: they are never a memory scope, recall never queries them directly, and there is no `agh memory promote --session` surface in Slice 1. Durable memory promotion happens only through extractor/controller/dreaming decisions and explicit `agh memory promote <filename>`.

- **Integration with workspace_id resolver** (ADR-004): `<workspace_id>` partitioning in `~/.agh/sessions/` survives workspace moves automatically. Workspace deletion via `agh memory reset --workspace X` purges both `<workspace>/.agh/agh.db` AND `~/.agh/sessions/<workspace_id_X>/` subtree.

- **Retention**:
  - Live events.db rows: kept until session ends + 24h grace period; then events.db rows for that session are purged (events live in JSONL).
  - JSONL files: **never hard-delete by default**. Cold-archived after 30 days to `~/.agh/sessions/_archive/YYYY-MM/<workspace_id>/<session_id>/`. Configurable via `[memory.session]` block (see Implementation Notes).
  - Hermes pattern (`prune_sessions(older_than_days=90)` idempotent auto-prune at 24h interval) is adopted: `agh sessions prune --older-than 90d` runs daily as a daemon-managed task.

## Alternatives Considered

### Alternative 1: events.db only (no JSONL)

- **Description**: Keep all session events in events.db forever. No filesystem ledger.
- **Pros**: Simplest. One write path. No materializer.
- **Cons**: events.db grows unbounded across years. No human-readable surface. No `tar` backup of "this session's history." Forensic queries require API access.
- **Why rejected**: User explicitly wants a per-session ledger they can read. events.db alone doesn't satisfy that need.

### Alternative 2: JSONL authoritative (Codex pattern)

- **Description**: Write directly to JSONL during the session; SQLite (events.db) is the index built from JSONL.
- **Pros**: Matches Codex shape. Filesystem is the durable store.
- **Cons**: Slower writes (no transaction grouping). Harder to query mid-session. AGH's existing events.db is well-tested transactional infrastructure; replacing it is a big change for unclear benefit.
- **Why rejected**: events.db is already the live truth in AGH; replacing it would invert a working design. The forensic JSONL satisfies the user's need without the inversion.

### Alternative 3: JSONL inside workspace folder (OpenFang shape)

- **Description**: `<workspace>/.agh/sessions/<session_id>/ledger.jsonl`.
- **Pros**: Co-located with workspace state. `mv` workspace moves the ledger too.
- **Cons**: Clutters workspace dir if `.agh/` is git-tracked. Doesn't work for unbound sessions (no workspace). Backups of sessions across workspaces require iterating each workspace's `.agh/`. Operators commonly want "all my sessions" listed in one place.
- **Why rejected**: 3 of 5 competitors put session ledgers in $HOME — that's the production-validated pattern. Workspace_id partitioning under `~/.agh/sessions/` gives the same lifecycle benefits without the clutter.

### Alternative 4: Per-day rollup (Codex YYYY/MM/DD subdirs)

- **Description**: `~/.agh/sessions/YYYY/MM/DD/rollout-<session_id>.jsonl`. Codex shape.
- **Pros**: Natural archival by date.
- **Cons**: Sessions belong to workspaces, not days. Operator question "show me all sessions in workspace X" requires walking all date dirs. Workspace_id partitioning answers that question directly.
- **Why rejected**: AGH operators reason in terms of workspaces, not dates. workspace_id partitioning is the natural primary key.

## Consequences

### Positive

- Live writes stay fast and transactional (events.db unchanged).
- Forensic surface satisfies the user's "I keep a ledger" intuition.
- Workspace_id partitioning makes "list all sessions in workspace X" trivial (`ls ~/.agh/sessions/<workspace_id>/`).
- Backup is easy: `tar ~/.agh/sessions/` ships every session ledger.
- JSONL is human-grepable; one line per event; standard tooling works.
- Workspace move doesn't affect ledger paths (workspace_id stable per ADR-004).
- Workspace deletion is clean (`agh memory reset --workspace X` purges both DB and ledger subtree).

### Negative

- Two storage paths per session. Mitigated by clear lifecycle: live → events.db; ended → JSONL; events.db rows purged 24h after session end.
- JSONL materializer is new code. ~150 LOC + tests. Atomic write via existing `fileutil.AtomicWrite`.
- Session that crashes mid-flight may have events in events.db without a corresponding JSONL. Recovery: daemon boot scans for sessions with `state = stopped` and no JSONL; runs materializer.

### Risks

- Materializer fails after session end → no forensic JSONL. Mitigation: materialization is idempotent; daemon retries on next boot. events.db rows survive until materialization succeeds (24h grace with retry).
- JSONL grows large for long sessions (hours of events). Mitigation: per-session JSONL is bounded by session length. Cold-archive at 30 days moves the file; `max_archive_bytes = 10 GiB` safety valve triggers oldest-first cold deletion.
- workspace_id changes (rare; e.g., manual `.agh/workspace.toml` edit) orphans the JSONL subtree. Mitigation: `agh sessions repair` re-keys orphaned subtrees by finding the matching workspace_id from `meta.json`.

## Implementation Notes

- New package: `internal/sessions/ledger`. Owns materializer, retention, and JSONL read API.
- `OnSessionEnd` hook handler runs the materializer asynchronously (detached context per SD-010).
- `events.db` schema unchanged; retention is operational (a daily prune task removes session-scoped rows older than 24h post-end where the corresponding JSONL exists).
- `agh sessions list [--workspace X]`, `agh sessions show <id>`, `agh sessions replay <id>`, `agh sessions prune --older-than <duration>`, `agh sessions repair` — all CLI verbs ship with v2.
- Config keys:

  ```toml
  [memory.session]
  ledger_format        = "jsonl"          # only option in v2
  ledger_root          = "~/.agh/sessions" # default
  events_purge_grace   = "24h"            # how long live events.db rows survive after JSONL materialization
  cold_archive_days    = 30
  hard_delete_days     = 0                # 0 = never; explicit prune via CLI
  max_archive_bytes    = 10737418240      # 10 GiB safety valve
  unbound_partition    = "_unbound"
  ```

- JSONL schema is contract-stable; `ledger_meta.version = 1` in Slice 1 and later additions are append-only (new optional fields). Documented in `internal/api/contract` and codegen-shipped to `web/src/generated/agh-openapi.d.ts`.
