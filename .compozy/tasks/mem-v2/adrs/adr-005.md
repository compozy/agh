# ADR-005: `_system/` Namespace Invariant — Never Injected into Prompt

## Status

Accepted

## Date

2026-05-04

## Context

The dreaming worker (ADR-010 forthcoming) writes consolidation outputs to disk for forensic browsing. The extractor records its decisions (ADD/UPDATE/NOOP/REJECT) for audit. The mid-session delta-note channel (Codex pattern, `analysis_codex.md` §1, §8.2 — `~/.codex/memories/extensions/ad_hoc/notes/<ts>-<slug>.md`) lets agents propose memory mutations without bypassing the controller.

If these system-generated artifacts get indexed into `MEMORY.md` and injected into the prompt, the model reads **its own dreaming synthesis** as if it were "user instruction." That's a feedback loop that amplifies whatever bias the synthesis introduced — a documented memory-poisoning risk (`analysis_ai-memory.md` §9.1).

Competitor patterns:

- **OpenClaw** writes dreaming outputs to `<workspaceDir>/memory/dreaming/{light,deep,rem}/YYYY-MM-DD.md` (`analysis_openclaw.md` §1.5). Dreaming output is **separate from `MEMORY.md`**; only promoted entries cross into `MEMORY.md` after passing scoring gates.
- **Codex** segregates the memory handbook (`MEMORY.md`, `memory_summary.md`) from the mutation channel (`extensions/ad_hoc/notes/`) and from the rollout summaries (`rollout_summaries/`). The `read_path.md` template (`analysis_codex.md` §1) hard-codes which files are quick-pass-eligible.
- **Claude Code** writes session JSONL to `~/.claude/projects/<sanitized-cwd>/<sessionId>.jsonl` — entirely separate from memdir's curated tree.

The convergent rule: **system-generated memory artifacts must live in a namespace that is structurally excluded from prompt injection.**

## Decision

Adopt **`_system/` as a per-scope invariant namespace** with the following rules:

- **Layout**: every scope root has a `_system/` subdirectory:

  ```
  <scope>/memory/_system/
  ├── dreaming/YYYYMMDD-<slug>.md       # consolidation synthesis output
  ├── extractor/YYYY-MM-DD.jsonl         # extractor decisions (ADD/UPDATE/NOOP/REJECT)
  ├── extractor/failures/<run_id>.md     # DLQ for failed consolidations
  └── ad_hoc/<iso8601>-<slug>.md         # delta notes from agents/operators
  ```

- **Invariant**: files under `_system/` are **never injected** into `MEMORY.md` or the system prompt. Enforced at three layers:
  1. **`Scan` walker**: extends `shouldSkipFile` (current `internal/memory/store.go:1173-1175`) to skip the `_system/` directory entirely. Recursive walks stop at `_system/`.
  2. **Catalog rows**: `_system/` artifacts get an `injection BOOLEAN NOT NULL DEFAULT 0` column. Recall pipeline filters `WHERE injection = 1` unless the caller passes `--include-system`.
  3. **Controller validation**: any mutation that would create a top-level `_system_*.md` (matching the pattern but not under `_system/`) is rejected with `MemoryError(KIND_RESERVED_NAMESPACE)`.

- **Subdir taxonomy** (each subdir has a defined owner and lifecycle):
  - `dreaming/` — owned by the dreaming worker (ADR-010). Output of LLM synthesis runs. Browsable via `agh memory dream show <date>`. Read by next dreaming run for cumulative learning.
  - `extractor/YYYY-MM-DD.jsonl` — append-only daily log of extractor decisions. One JSONL line per decision with full provenance (input candidate, top-K dedup matches, decision, why-this-decision metadata). Mirrors `memory_events` for human grepping.
  - `extractor/failures/<run_id>.md` — DLQ for failed consolidations (goclaw lesson, `analysis_goclaw-paperclip-multica.md` §1.4). Manual retry: `agh memory dream retry <run_id>`.
  - `ad_hoc/<iso8601>-<slug>.md` — Codex-style delta notes. **Only place agents may write outside controller-mediated curated entries.** Native tool `agh__memory_note(scope, content, slug?)` is the only write surface (ADR-009 forthcoming). Next dreaming run reconciles ad_hoc notes into curated entries; processed notes are deleted.

- **`agh memory list --include-system`** and **`agh memory show <path> --include-system`** allow operators to inspect `_system/` content explicitly. Default lists hide it.
- **`agh memory reset`** preserves `_system/` by default (forensic friendly). `--include-system` purges it.

## Alternatives Considered

### Alternative 1: Filename prefix convention (e.g., `_dreaming_*.md`)

- **Description**: Reserve underscore-prefixed filenames for system artifacts; no separate directory.
- **Pros**: Flat directory layout, simpler.
- **Cons**: Filename pattern matching is fragile — operators may accidentally name a file `_my_notes.md`. No structural enforcement at the filesystem layer. `agh memory list` requires regex filtering instead of dir skip.
- **Why rejected**: Convention-only enforcement violates the safety-by-default principle. Directory boundary is unambiguous.

### Alternative 2: Out-of-tree storage (e.g., `<scope_root>/dreaming/` peer to `memory/`)

- **Description**: Move dreaming and extractor outputs out of `memory/` entirely, to sibling directories.
- **Pros**: Maximum separation; `memory/` is purely curated content.
- **Cons**: Loses scope-locality (dreaming output is per-scope, so it logically belongs inside the scope's tree). Multiplies the directory count operators must remember. `agh memory dream show` becomes `agh dream show` (separate verb namespace).
- **Why rejected**: `_system/` keeps dreaming/extractor output in the right scope; the underscore prefix signals "machine-managed" without breaking scope locality.

### Alternative 3: Database-only system artifacts (no files)

- **Description**: Store dreaming synthesis, extractor decisions, and ad_hoc notes as event-log rows; no filesystem footprint.
- **Pros**: No file lifecycle management. No GC of stale `_system/` files.
- **Cons**: Loses the human-grepability of operations. Codex's `extensions/ad_hoc/notes/` is filesystem because operators need to read and edit it. Dreaming output as plain Markdown is far more debuggable than DB rows.
- **Why rejected**: Filesystem grepability is a core AGH posture (CLI-first, agent-first). Operators must be able to `cat <scope>/memory/_system/dreaming/YYYYMMDD-*.md`.

## Consequences

### Positive

- Structural enforcement: filesystem boundary makes the rule unambiguous.
- Dreaming + extractor + ad_hoc all have a stable home with a defined lifecycle.
- Memory-poisoning feedback loop is impossible by construction (system-generated content can't reach the prompt without explicit operator opt-in).
- Forensic surface: `agh memory dream show <date>`, `agh memory extractor show <date>`, `agh memory adhoc list` all read from `_system/` directly.
- Consistent across all scopes (`global`, `workspace`, `agent-workspace`, `agent-global`) — same `_system/` subdir wherever a memory tree exists.

### Negative

- Reserved namespace adds a sharp edge for operators: a file named `_system_notes.md` at the top level of `<scope>/memory/` is rejected. Mitigated by clear error message and CLI-side validation.
- Directory walk needs awareness of `_system/`. Mitigated by single check in `shouldSkipFile`.
- `agh memory list --include-system` is required for operators to see system artifacts. Default-hidden behavior may surprise some users; mitigated by docs and the `--include-system` shorthand.

### Risks

- Bug: a controller code path that forgets the `injection = 0` filter could leak `_system/` rows into recall. Mitigation: integration test in `agh-test-conventions` that asserts `Recall` never returns `_system/` rows without `--include-system`. Unit test on `Scan` skip behavior.
- Operator who writes their own files into `_system/` directly bypasses the controller. Mitigation: tolerated; `_system/` is operator-readable and operator-writable; if someone wants to hand-craft a dreaming output, that's intentional.

## Implementation Notes

- `internal/memory/store.go::Scan` extends `shouldSkipFile` to skip directory `_system`.
- `memory_catalog_entries` schema migration adds `injection BOOLEAN NOT NULL DEFAULT 1`. Files under `_system/` get `0` on indexing.
- `Recall.candidates(...)` and `Assembler.LoadPromptIndex(...)` filter `WHERE injection = 1` unless `IncludeSystem = true` in opts.
- `Controller.Decide` rejects writes whose target path (after sanitization) lives under `_system/` unless the caller is the dreaming worker, extractor, or ad_hoc native tool — identified by `WriteSource` enum (`OPERATOR | EXTRACTOR | DREAMING | AD_HOC | EXTENSION`).
- `agh memory list`, `agh memory show`, `agh memory search`, `agh memory reindex` add `--include-system` flag.
- New verb: `agh memory dream show <date>`, `agh memory dream retry <run_id>`.
- New verb: `agh memory adhoc list`, `agh memory adhoc show <slug>`.
