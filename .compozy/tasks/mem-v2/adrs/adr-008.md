# ADR-008: MemoryProvider Extension ABC — Hermes 10-Hook Lifecycle

## Status

Accepted

## Date

2026-05-04

## Context

AGH today has speculative memory-extension types declared in `internal/memory/types.go` but **unconsumed by the daemon**:

- `ProviderHookRunner`, `ProviderHookRequest`, `ProviderHookResult`, `ProviderHookEvent`

`analysis_agh-current.md` §16 flagged this as an SD-009 violation: "data exists / consumer missing." The integration map (`analysis_integration-map.md`) initially recommended deleting these types and re-authoring under v2 packages. That recommendation was wrong — these types are exactly the seam that should grow into the production provider contract, not be replaced.

The strongest production reference is **Hermes' `MemoryProvider` ABC** (`analysis_hermes.md` §1, §3): a 10-method lifecycle interface with rich metadata-aware dispatch, single-external-provider invariant, and curator-mediated state. Hermes' real moat over Claude Code / Codex is exactly this extensibility surface: third-party memory providers (Honcho, Mem0, Hindsight) plug in via the same ABC the bundled local provider implements.

AGH's posture (`internal/CLAUDE.md`):

- Designed for incremental extension — new capabilities arrive as new packages without modifying existing ones.
- Every feature must be extensible via extension surfaces and agent-manageable (SD-011).
- Daemon must actually delegate to enabled providers (otherwise SD-009 violation persists).

Three modeling options were considered:

1. Extend declared types into the full Hermes 10-hook ABC.
2. Adopt a curated subset (4 hooks: Initialize / Recall / OnMemoryWrite / Shutdown), defer the rest to slice 4.
3. Defer extension API entirely; slice 1 ships in-process only.

The user selected option 1.

## Decision

Adopt the **Hermes-style 10-method `MemoryProvider` ABC** in `internal/memory/contract`, extending the existing declared types in `internal/memory/types.go`:

```go
// MemoryProvider is the lifecycle interface for any memory backend that
// can be plugged into AGH (bundled or extension-supplied).
type MemoryProvider interface {
    Initialize(ctx context.Context, init ProviderInit) error

    SystemPromptBlock(ctx context.Context, req SnapshotRequest) (SnapshotResult, error)
    Recall(ctx context.Context, req RecallRequest) (RecallResult, error)        // optional; nil-safe
    Prefetch(ctx context.Context, req PrefetchRequest) error                    // optional
    SyncTurn(ctx context.Context, rec TurnRecord) error                         // optional

    OnSessionEnd(ctx context.Context, rec SessionEndRecord) error
    OnSessionSwitch(ctx context.Context, rec SessionSwitchRecord) error
    OnPreCompress(ctx context.Context, req PreCompressRequest) (PreCompressHint, error)
    OnMemoryWrite(ctx context.Context, rec WriteRecord) error

    Shutdown(ctx context.Context) error
}
```

Key invariants:

- **Single active provider per workspace.** Hermes invariant; prevents merge-conflict semantics. Selected via `[memory.provider]` block in `config.toml`.
- **Bundled local provider** (`internal/memory/provider/local`) is the reference implementation. Wraps the existing SQLite catalog + Markdown store. Always present as the default.
- **Daemon delegation is mandatory.** The composition root (`internal/daemon/`) routes every memory operation through the active provider. No more "declared but unconsumed" types — fixes the SD-009 violation called out in `analysis_agh-current.md` §16.
- **Optional methods** (Recall, Prefetch, SyncTurn) accept nil-safe defaults. A provider that does not implement Recall falls back to the bundled local provider's Recall against the catalog.
- **Provider-supplied tools dedup against built-in tool names** (Hermes pattern, `analysis_hermes.md` §1). Collisions log + reject at register-time.
- **Metadata-aware dispatch via opts structs** (Go form of Hermes' Python signature-introspection): every method takes a typed request struct, never positional args, so v2.1+ can add fields without breaking implementations.
- **Lifecycle ordering**: `Initialize` (per-workspace, per-daemon-lifetime) → on `on_session_started` hook fires `SystemPromptBlock` → repeated turn cycle (`Prefetch`, `Recall`, `SyncTurn`, `OnPreCompress` if compaction fires) → on `on_session_stopped` hook fires `OnSessionEnd` / `OnSessionSwitch` → `Shutdown` (per-workspace at daemon shutdown). Canonical lifecycle table in TechSpec §Hooks (peer-review N13 — there is no `OnSessionStart` method; the start trigger is the `on_session_started` hook event firing `SystemPromptBlock`).

## Alternatives Considered

### Alternative 1: Curated 4-hook subset

- **Description**: Initialize / Recall / OnMemoryWrite / Shutdown only. Defer the other 6 to slice 4.
- **Pros**: Smaller surface to stabilize. Faster slice 1 delivery.
- **Cons**: Adding hooks later is a breaking change for any extension that adopted the 4-hook contract. The extra 6 hooks are exactly what makes Hermes' contract production-grade — `SystemPromptBlock` for cache-stable prompt injection, `OnPreCompress` for pre-compaction flush, `OnSessionSwitch` for split-session integrity. Cutting them ships a less competitive contract.
- **Why rejected**: User chose to ship the full ABC in slice 1; the 6 "deferred" hooks are AGH's competitive moat against Claude Code / Codex.

### Alternative 2: Defer extension API entirely

- **Description**: Slice 1 has no extension API. Memory is in-process only. Delete the declared types.
- **Pros**: Smallest slice 1 scope.
- **Cons**: Hermes' real moat is the provider-plugin model. Deferring it means slice 1 ships at parity with Claude Code (no extension API). Subsequent slices then must invent the ABC from scratch when external providers (Mem0, Honcho, Hindsight) are added. The greenfield rule against re-author churn applies.
- **Why rejected**: User explicitly stated AGH must match-or-exceed Hermes; deferring the provider contract concedes the most competitive surface.

### Alternative 3: Adopt Letta or Mem0's API verbatim

- **Description**: Use Letta's three-tier memory API or Mem0's add/search/update/delete shape as the extension contract.
- **Pros**: Industry-recognized shape; could adopt their existing tooling.
- **Cons**: Letta and Mem0 are products, not contracts. Their APIs encode product decisions (Letta's three-tier model; Mem0's specific dedup logic). Adopting them constrains AGH to their evolution. Hermes' ABC is a cleaner protocol because Hermes is itself a runtime, not a product surface.
- **Why rejected**: Hermes' contract is the production-validated runtime-level shape. Letta and Mem0 ship as providers under it.

## Consequences

### Positive

- AGH's competitive moat: any third-party memory provider (Mem0, Honcho, Hindsight, Letta, custom) implements one Go interface and plugs in.
- Reuses the declared `Provider*` types in `internal/memory/types.go` instead of deleting them — corrects the integration-map's "delete and re-author" call.
- Daemon delegation closes the SD-009 violation that has been outstanding.
- Bundled local provider ships as the reference implementation; new providers can crib from it.
- Optional methods give providers flexibility (a write-through provider may not implement Recall; a read-only cache provider may not implement OnMemoryWrite).
- Per-workspace provider selection allows different workspaces to use different providers (workspace A → local; workspace B → Mem0-backed).

### Negative

- 10 methods is a lot of surface for v1. Mitigated by the Optional flag and the bundled local provider as a copy-and-adapt template.
- Single-active-provider invariant means an operator can't run two providers in parallel for migration. Mitigated by the bundled local provider being always-present as the fallback / reference.
- Provider failures (timeout, crash) need fail-open semantics so the daemon doesn't lock up. Mitigated by per-method timeouts and circuit-breaker pattern (see Implementation Notes).

### Risks

- A poorly-implemented external provider could write to its own store but fail OnMemoryWrite, losing audit. Mitigation: OnMemoryWrite is mandatory, not optional. Audit event written even when provider's mutation succeeds.
- A provider that violates the prompt-cache invariant (mutating SystemPromptBlock mid-session) breaks ADR-001's frozen snapshot. Mitigation: SystemPromptBlock result is captured once at session boot; mid-session calls return cached value.
- Provider tools collide with built-in tools (e.g., Mem0 ships its own `memory_search` tool while AGH ships `agh__memory_search`). Mitigation: register-time dedup in `internal/tools/registry`; collision rejects the provider's tool with a structured error logged at boot.

## Implementation Notes

- New package: `internal/memory/contract` defines `MemoryProvider` interface + all opts/result types.
- New package: `internal/memory/provider/local` implements the bundled local provider against the existing catalog + Markdown store.
- `internal/memory/types.go` is the source inventory for the existing declared provider shapes, but Slice 1 hard-cuts every caller to `internal/memory/contract`. After the DTO move, the old file is deleted rather than left behind as a shim.
- `internal/extension/host_api.go` extends the host API with `MemoryProviderRegistry` interface for extension-supplied providers.
- Extension contract delivery: providers register via `MemoryProviderManifest{ name, version, capabilities }` in extension manifests (`internal/extension/contract`).
- Per-method timeout + circuit breaker: `[memory.provider.<name>]` block with `timeout_ms`, `failure_threshold`, `cooldown_seconds`. Circuit-breaker open → daemon falls back to bundled local provider for that method.
- Codegen co-ship: `agh-contract-codegen-coship` skill applies because new types cross `internal/api/contract`.

## Post-Approval Refinement (Codex peer-review B4 incorporation, 2026-05-04)

The original draft put `MemoryProvider` and its DTOs in `internal/memory/contract` and described it as "depending on nothing in `internal/memory`" — but the interface signatures referenced `controller.Decision`, `controller.Candidate`, recall-package types, and extractor-package `TurnRecord`. That coupled extension authors to internal AGH packages and risked import cycles.

**Refinement adopted:**

1. **All shared DTOs move to `internal/memory/contract`.** Concretely: `Scope`, `AgentTier`, `Origin`, `Type`, `Header`, `Provenance`, `Op`, `DecisionSource`, `Candidate`, `Decision`, `Query`, `RecallOptions`, `Packaged`, `Block`, `PackagedEntry`, `Trigger`, `TurnRecord`, `TranscriptSnapshot`, `WriteRecord`, `ProviderInit`, `SnapshotRequest`, `SnapshotResult`, `RecallRequest`, `RecallResult`, `PrefetchRequest`, `PreCompressRequest`, `PreCompressHint`, `SessionEndRecord`, `SessionSwitchRecord`, plus `MemoryProvider` itself.

2. **`internal/memory/contract` depends only on stdlib + `internal/logger`.** Imports from `controller`/`recall`/`extractor`/`store` are explicitly forbidden by the boundary check in `mage Boundaries`.

3. **All concrete packages depend on `contract`, not the reverse.** `controller.Decide` returns `contract.Decision`, `recall.Recall` returns `contract.Packaged`, `extractor.Extract` returns `[]contract.Candidate`. Provider implementations import only `contract`.

4. **`internal/memory/types.go` is deleted in the same slice** after callers move to `internal/memory/contract`. Greenfield discipline forbids leaving a re-export shim behind.

This makes the SDK boundary stable: an extension author writing a custom `MemoryProvider` imports one package (`internal/memory/contract`), and that package's churn is bounded to enum and DTO additions, not to controller-internal evolution.

The 10-hook lifecycle, the single-active invariant, the bundled-local-as-reference pattern, and the daemon-delegation-mandatory rule are all unchanged.
