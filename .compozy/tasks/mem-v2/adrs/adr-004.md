# ADR-004: Stable Workspace ID via .agh/workspace.toml

## Status

Accepted

## Date

2026-05-04

## Context

AGH today keys workspace-scope memory rows by `workspace_root TEXT` — the **filesystem path** to the workspace (`internal/memory/catalog.go:1260-1270` `deriveWorkspaceRoot`). This is a documented bug:

> Moving a workspace (`mv ~/dev/projeto-a ~/projects/projeto-a`) orphans all its memory catalog rows. Subsequent reads return empty; writes create a duplicate set under the new path; the old set lingers indefinitely. (`analysis_agh-current.md` §3, §16.)

The bug is silent — operators see "empty memory" with no error. There is no rollback path because the orphaned rows are valid, just unreachable from the new path.

Beyond the move scenario, path-keyed identity also breaks:

- Symlinked workspaces (`~/work` → `~/Documents/work`) — same workspace, two paths.
- Worktrees (Claude Code uses `findCanonicalGitRoot` — `analysis_claude-code.md` §5 — to dedupe sibling worktrees).
- Containerized workspaces where the host path differs from the container path.
- Machine moves where home dir relocates.

The fix is universal: separate **workspace identity** (a stable token) from **workspace location** (a path that can change).

Competitor patterns:

- **Claude Code** uses canonical git root via `findCanonicalGitRoot` (`analysis_claude-code.md` §5) — works for git-backed workspaces only.
- **Hermes** uses profile-aware `HERMES_HOME` (`hermes_cli/main.py:94-163`) — manual operator binding, not workspace-bound.
- **OpenClaw** uses agentId, not workspaceId — partition is per-agent.
- **Codex** sanitizes cwd to a directory name (`~/.codex/sessions/YYYY/MM/DD/`) — date-keyed, not workspace-keyed.

AGH must support **non-git workspaces** (operators run AGH in any directory). A git-only solution is insufficient.

## Decision

Adopt **stable workspace_id ULID stored in `<workspace>/.agh/workspace.toml`** (peer-review N11 — UUID alternative rejected; ULID Crockford base32 is the canonical format):

- On first workspace-scope memory mutation (or first daemon touch of the workspace), the resolver creates `<workspace>/.agh/workspace.toml` with:

  ```toml
  workspace_id = "01HXJ9YR4Q..."  # ULID (Crockford base32, 26 chars; peer-review N11 — UUIDv7 alternative rejected)
  created_at   = "2026-05-04T14:30:00Z"
  realpath_at_creation = "/Users/pedronauck/dev/projeto-a"  # informational only
  ```

- **All catalog rows, event log entries, recall signals, consolidation runs, session ledgers, and per-workspace DB filenames key on `workspace_id`** — never on the path.
- **Workspace move** (`mv path-A path-B`): no action required. `<workspace>/.agh/workspace.toml` moves with the directory; subsequent daemon access reads the same ULID; all rows still key correctly.
- **Path resolver** (`internal/workspace.Resolver.Resolve(ctx, workspace_ref)`): walks ancestors looking for `.agh/workspace.toml`; reads ULID; returns `(workspace_id, realpath)`. Updates global `workspace_index` table (in `$AGH_HOME/agh.db`) with `(workspace_id, realpath, last_seen_at)` on every resolution.
- **`agh memory edit` / write CLI** triggers lazy creation if `.agh/workspace.toml` is missing.
- **`.agh/workspace.toml` is committable to git** (the ULID is workspace identity; multiple operators on the same repo share it). For `.agh/workspace.toml` to be useful across machines, it must be tracked. AGH's docs will recommend committing it; operators who don't want to share workspace_id may add it to `.gitignore` and accept that team members will have separate workspace_ids for the same repo.

## Alternatives Considered

### Alternative 1: Derive workspace_id from canonical git root

- **Description**: `workspace_id = SHA-256(canonical-git-root-realpath)[:16]`. Claude Code-style.
- **Pros**: No file to create. Deterministic. Identical across machines (if the git remote URL or canonical path matches).
- **Cons**: Doesn't work for non-git workspaces. Fails for monorepos where multiple subdirs are independent workspaces. Path collisions across distinct git roots that resolve to the same canonical path (rare but possible).
- **Why rejected**: AGH cannot assume git presence. Non-git workspace must be a first-class case.

### Alternative 2: Path hash (no file, no git)

- **Description**: `workspace_id = SHA-256(realpath)[:16]`.
- **Pros**: No file to create. No git dependency.
- **Cons**: Re-introduces the path-bug — moving the workspace changes the hash. Doesn't solve the problem.
- **Why rejected**: Defeats the purpose.

### Alternative 3: ULID stored in `.agh/workspace.toml` (chosen)

- **Description**: Stable ULID (Crockford base32, 26 chars, sortable by creation time) in a config file inside the workspace. Peer-review N11 confirmed ULID over UUIDv7.
- **Pros**: Survives moves. Works for non-git. Operator can inspect / commit / git-share. Forward-compatible (file can hold other workspace metadata later).
- **Cons**: Requires creating a new file in workspaces that don't have one. Operators may delete it or forget to commit it.
- **Why chosen**: Universal across workspace types. Handles the move case and the non-git case. File creation is one-shot at first touch.

### Alternative 4: UUID stored in global registry only

- **Description**: `$AGH_HOME/workspaces.toml` maps `realpath → workspace_id`. No file inside the workspace.
- **Pros**: No clutter in the workspace.
- **Cons**: Re-introduces the path-bug at lookup time — moving the workspace changes the realpath, which changes the registry key, which invalidates the mapping. Could be mitigated with file-content fingerprinting, but that's lossy.
- **Why rejected**: The whole point of stable identity is to decouple from path. A path-keyed registry is just a renamed path-keying.

## Consequences

### Positive

- `mv` workspace doesn't orphan memory — bug fixed.
- Symlinked / containerized / migrated workspaces all key correctly.
- Per-workspace DB path (ADR-003) keys on `workspace_id` cleanly.
- Forward-compatible: `.agh/workspace.toml` can grow other fields (workspace name, owner, timezone, default scope, etc.) without schema changes.
- Operators who commit `.agh/workspace.toml` get team-wide stable workspace identity (multiple operators share the same memory rows).

### Negative

- New file in every workspace. Operators must accept it; AGH docs will recommend committing it.
- Daemon migration: existing workspaces have no `workspace.toml`. Migration generates one on first v2-touch and backfills `workspace_id` into existing catalog rows.
- File deletion by accident orphans the workspace (now seen as "new"). Mitigation: backup file in `$AGH_HOME/workspaces/<workspace_id>.toml.bak` mirrors the canonical file; recovery via `agh workspace recover --realpath <path>` matches by old realpath and rewrites the file.

### Risks

- Two operators on the same repo, neither commits `workspace.toml`: each gets their own workspace_id; their memory doesn't merge. Mitigation: docs guide; on first creation, AGH prints "tip: commit `.agh/workspace.toml` to share workspace identity with your team."
- Workspace cloned to a new path with `.agh/workspace.toml` intact: both copies share the same workspace_id but diverge in DB content. Mitigation: `.agh/agh.db` is also copied with the directory, so divergence requires writes after the copy. AGH detects multi-realpath usage of the same workspace_id and warns: "workspace 01HXJ... seen at two paths; consider regenerating ULID for the duplicate."
- Permission-denied on `.agh/workspace.toml` creation (read-only mount, etc.): **fatal init error** (peer-review safety #8 + cross-cutting #5). The daemon refuses to operate on a workspace it cannot persist identity for. **Superseded** the earlier ephemeral-fallback design — silent identity fork is unacceptable.

## Implementation Notes

- File format: TOML, single section, ~5 keys initially.
- ULID is the chosen format (Crockford base32, 26 chars). UUIDv7 was considered and rejected for parser/validator simplicity; ULID is sortable by creation time and visually distinguishable in logs (peer-review N11).
- Resolver: `internal/workspace.Resolver.Resolve(ctx, ref)` returns `ResolvedWorkspace { ID, Realpath, CreatedAt }`. Already exists for path resolution; v2 extends to also load / create `workspace.toml`.
- `agh workspace info`, `agh workspace list`, `agh workspace recover` CLI verbs ship with v2.
- Migration: schema bump in `memory_catalog_entries` adds `workspace_id TEXT NOT NULL`; backfill walks existing rows by `workspace_root` path, resolves to ULID via on-the-fly `workspace.toml` creation, updates rows. Numbered migration in registry (`agh-schema-migration` skill mandatory).
- `deriveWorkspaceRoot(path)` (current code at `internal/memory/catalog.go:1260-1270`) is **deleted**, not bridged. Greenfield rule.
