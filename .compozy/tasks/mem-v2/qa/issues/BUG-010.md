# BUG-010: UDS observe parity expected the pre-Memory-v2 augmenter contract

## Status

Fixed

## Severity / Priority

- Severity: Medium
- Priority: P2

## Scope

- Scenario: HTTP/UDS observe parity integration.
- Surface: UDS transport parity test contract.
- CLI/Web parity affected: UDS parity validation was blocking final runtime E2E.
- Browser tool: Not involved.
- Live provider/LLM involved: No.
- Environment: Runtime E2E final-postfix QA run.

## Behavioral Impact

The UDS transport parity test still expected only the pre-Memory-v2 prompt augmenters, so valid Memory v2 runtime events failed parity validation.

## Reproduction

1. Run `go test -race -tags integration ./internal/api/udsapi -run 'TestUDSTransportObserveHarnessLifecycleParityMatchesHTTP' -count=1`.
2. Observe the expected augmenter count mismatch.

## Expected

UDS observe parity includes Memory v2 durable memory, skills, and situation augmenters, matching HTTP lifecycle events.

## Actual

The test expected two augmenters instead of the three now emitted by the Memory v2 harness contract.

## Root Cause

`internal/api/udsapi/transport_parity_integration_test.go` had not been updated for the Memory v2 prompt augmentation contract.

## Fix

Updated the UDS parity test expectation to assert all three Memory v2 harness `harness.augmenter_applied` events.
