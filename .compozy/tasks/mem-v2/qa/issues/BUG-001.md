# BUG-001: Public Memory transports cannot resolve stable workspace identity

## Status

Fixed

## Severity

P0 / Critical

## Scenario

TC-SCEN-001: controller-backed Memory writes must be visible to CLI, HTTP, UDS, and native tool reads/search without an undocumented reindex.

## Reproduction

1. Start the isolated daemon with the lab `AGH_HOME`, `AGH_HTTP_PORT`, and `AGH_UDS_PATH`.
2. Write workspace-scoped memory with `agh memory write --workspace <lab> --scope workspace --type project --name qa-search-visibility --content @qa/fixtures/search-visibility.md -o json`.
3. Confirm immediate CLI search succeeds with `agh memory search "search visibility sentinel" --workspace <lab> --scope workspace -o json`.
4. Read `<lab>/.agh/workspace.toml` and use its `workspace_id` value in the HTTP and UDS memory search request body.
5. Register the workspace with `agh workspace add <lab> --name mem-v2-qa -o json`.
6. Repeat the HTTP and UDS memory search request using the stable `workspace_id`.

## Expected

The stable workspace identity from `.agh/workspace.toml` resolves after the workspace is registered, and HTTP/UDS memory search returns the same result as CLI search without requiring a reindex or absolute-path fallback.

## Actual

CLI search succeeds immediately. HTTP and UDS search succeed only when the absolute workspace path is placed in the `workspace_id` request field. Requests using the stable workspace identity fail before and after `workspace add`:

- HTTP returns `{"code":"memory.internal","message":"Internal Server Error"}`.
- UDS returns `{"code":"memory.internal","message":"workspace: lookup workspace by name \"01KQX3HAT21RHVV7MWA104FEJG\": workspace not found"}`.

## Root Cause

`workspace.Resolver.lookupWorkspace` treats unknown non-`ws_` identifiers as workspace names. It recognizes registered workspace IDs (`ws_...`), absolute paths, and names, but not the stable workspace ULID persisted in `.agh/workspace.toml`. Memory v2 indexes and public evidence expose the stable workspace identity, so HTTP/UDS clients cannot round-trip that identity back into the resolver.

## Fix Plan

Teach the workspace resolver to recognize canonical stable workspace ULIDs and resolve them by scanning registered workspaces' `.agh/workspace.toml` identities. Add regression tests proving a registered workspace can be resolved by the stable identity and that unknown stable identities fail with `ErrWorkspaceNotFound`.
