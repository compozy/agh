# BUG-011: Resource kernel write transactions surfaced SQLITE_BUSY during browser runtime seeding

## Status

Fixed

## Severity / Priority

- Severity: High
- Priority: P1

## Scope

- Scenario: Browser automation and tasks daemon-served E2E.
- Surface: Resource kernel SQLite writes during workspace auto-register change hooks.
- CLI/Web parity affected: Yes, web E2E workspace/session/task/automation state failed to seed through public HTTP APIs.
- Browser tool: Playwright daemon-served browser lane.
- Live provider/LLM involved: No.
- Environment: `make test-e2e-web` final-postfix run.

## Behavioral Impact

Concurrent browser runtime startup and workspace resolution could fail with `database is locked`, causing automation trigger setup to return 500 and tasks UI setup to retain stale workspace/session state.

## Reproduction

1. Run `make test-e2e-web`.
2. Observe `web/e2e/automation.spec.ts` fail seeding `POST /api/automation/jobs/:id/trigger` with HTTP 500.
3. Observe `web/e2e/tasks.spec.ts` fail because task creation remained visible after `POST /api/tasks` returned 404 from stale workspace/session state.
4. Inspect the daemon log for `resources: insert record "agent"/...: database is locked (5) (SQLITE_BUSY)`.

## Expected

Resource writes that participate in workspace auto-register hooks use the same bounded `BEGIN IMMEDIATE` retry discipline as other AGH SQLite writes.

## Actual

`Kernel.PutRaw` and `Kernel.DeleteRaw` used deferred `db.BeginTx`, and source-session/snapshot writes used a manual `BEGIN IMMEDIATE` without the shared busy retry policy.

## Root Cause

The resource kernel bypassed `store.ExecuteWrite`, so SQLITE_BUSY returned directly to workspace/session/automation paths during concurrent browser seeding instead of retrying for the short write-lock window.

## Fix

`internal/resources.Kernel` now routes `PutRaw`, `DeleteRaw`, and source-session/snapshot transactions through `store.ExecuteWrite`. The transaction callback still uses the existing `sqlExecutor` seam, preserving the resource kernel contract while inheriting bounded busy retries.
