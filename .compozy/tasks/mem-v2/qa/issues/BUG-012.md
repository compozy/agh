# BUG-012: Bridge E2E used an ambiguous global Close button selector

## Status

Fixed

## Severity / Priority

- Severity: Medium
- Priority: P2

## Scope

- Scenario: Bridge operator browser E2E.
- Surface: Playwright bridge test selector contract.
- CLI/Web parity affected: No production behavior changed, but the browser E2E lane was blocked.
- Browser tool: Playwright daemon-served browser lane.
- Live provider/LLM involved: No.
- Environment: `make test-e2e-web` final-postfix run.

## Behavioral Impact

The bridge test attempted to close the delivery-result dialog with a global `getByRole("button", { name: "Close" })`. Toast close buttons also matched, causing Playwright strict-mode failure before the test could verify bridge ingress health updates.

## Reproduction

1. Run `make test-e2e-web`.
2. In `web/e2e/bridges.spec.ts`, reach the test delivery result dialog.
3. Observe strict-mode violation because the global selector matches multiple toast close buttons plus the dialog close button.

## Expected

Browser E2E selectors target the operator-visible control under the intended dialog.

## Actual

The global selector matched five buttons, making the test nondeterministic and invalid under strict mode.

## Root Cause

The selector was not scoped to `bridgeUI.testDeliveryDialog`, even though the dialog locator already existed in the bridge selector fixture.

## Fix

`web/e2e/bridges.spec.ts` now clicks `bridgeUI.testDeliveryDialog.getByRole("button", { name: "Close" })`.
