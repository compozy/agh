# BUG-006: ACP mock fixtures did not match augmented Memory v2 prompts

## Status

Fixed

## Severity / Priority

- Severity: High
- Priority: P1

## Scope

- Scenario: Runtime E2E ACP mock fault and bridge flows.
- Surface: ACP mock driver, runtime E2E fixtures, prompt augmentation.
- CLI/Web parity affected: No public API shape changed; E2E contract was blocking final validation.
- Browser tool: Playwright exercised downstream flows after the fix.
- Live provider/LLM involved: No.
- Environment: Task 26 final-postfix QA run.

## Behavioral Impact

Memory v2 prompt augmentation adds situation, skills, durable memory, and bridge ingress envelopes before the user text. ACP mock fixtures matched raw prompts too literally, so realistic runtime E2E prompts failed even though the operator intent was correct.

## Reproduction

1. Run the runtime E2E lane after Memory v2 prompt augmentation is enabled.
2. Observe ACP mock failures where the fixture reports no matched turn for prompts wrapped by AGH situation, skills, durable memory, or bridge ingress context.

## Expected

Fixture matching canonicalizes AGH-owned prompt augmentation and matches on the user text contract.

## Actual

The mock agent rejected valid prompts because the raw prompt contained Memory v2 and bridge context wrappers.

## Root Cause

The ACP mock canonicalization logic did not strip all prompt augmentation layers introduced by Memory v2 and bridge ingress.

## Fix

`internal/testutil/acpmock/fixture.go` now strips AGH situation context, skills catalogs, durable memory recall blocks, and bridge inbound prompt envelopes before matching. No-match diagnostics now include both canonical and raw prompt forms.

## Regression Coverage

- `internal/testutil/acpmock/fixture_test.go` covers canonical user text stripping for prompt augmentation layers.
- `internal/testutil/acpmock/cmd/acpmock-driver/main_test.go` covers unmatched prompts not consuming occurrence counts.
