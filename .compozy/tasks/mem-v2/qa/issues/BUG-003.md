# BUG-003: Memory search API hid curated filenames and skipped realistic two-token recall queries

## Status

Fixed

## Severity

P0 / Critical

## Scenario

TC-SCEN-001 and TC-INT-003: controller-backed Memory writes must be discoverable through CLI, HTTP, UDS, and native search without identity drift, and public search payloads must expose the curated memory filename rather than internal chunk identifiers.

## Reproduction

1. Write a workspace memory with `agh memory write --scope workspace --type project --name qa-search-visibility-final --content @qa/fixtures/search-visibility.md -o json`.
2. Search the memory through CLI, HTTP, and UDS using the stable workspace identity.
3. Run the Memory runtime E2E lane: `go test -race -tags integration ./internal/daemon -run 'TestDaemonE2EMemoryCatalogCLIHTTPParityAndLegacyPathIsolation' -count=1`.
4. Run the recall-focused package tests for short natural queries.

## Expected

- Search payloads expose `memory.filename = project_qa_search_visibility_final.md`, `memory.type = project`, and the stable `workspace_id`.
- Two meaningful ASCII terms such as `auth sessions` are not treated as trivial when they are the operator's complete search intent.

## Actual

- HTTP and UDS search returned the internal chunk key as the public filename: `workspace::<workspace_id>::project_qa_search_visibility_final.md::chunk:0001`.
- The two-token query used by the runtime E2E path was skipped as trivial, which made a realistic Memory v2 recall/search path appear empty unless the query was padded.

## Root Cause

`PackagedEntry` carried only the recall chunk ID. The API conversion path therefore used the chunk ID as `MemoryEntrySummaryPayload.Filename`, losing the curated memory filename, type, and workspace identity needed by public search payloads. Separately, the recall trivial-query guard required three meaningful ASCII tokens, which rejected common two-term operator queries that the deterministic recall engine can score safely.

## Fix

- `internal/memory/contract/types.go` extends `PackagedEntry` with `filename`, `type`, and `workspace_id`.
- `internal/memory/recall/recall.go` populates those fields from catalog metadata and lowers the meaningful ASCII query floor from three tokens to two while preserving the non-ASCII guard.
- `internal/api/core/memory.go` maps public search result filenames, types, and workspace IDs from the packaged entry metadata instead of the chunk ID.
- `internal/memory/recall/recall_test.go` covers a two-token `auth sessions` query.
- Generated OpenAPI and TypeScript contracts were refreshed with `make codegen`.
