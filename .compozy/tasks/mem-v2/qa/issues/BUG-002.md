# BUG-002: Concurrent native Memory writes can expose atomic temp files to controller targeting

## Status

Fixed

## Severity

P1 / High

## Scenario

TC-INT-003: native Memory tools must route mutations through the Memory v2 controller without leaking storage implementation artifacts into decision targeting.

## Environment

- QA lab: `/Users/pedronauck/dev/qa-labs/agh-mem-v2-20260505-221648-724447-lab`
- Manifest: `/Users/pedronauck/dev/qa-labs/agh-mem-v2-20260505-221648-724447-lab/qa-artifacts/qa/bootstrap-manifest.json`
- `AGH_HOME`: `/Users/pedronauck/dev/qa-labs/agh-mem-v2-20260505-221648-724447-lab/.agh/runtime`
- HTTP base URL: `http://127.0.0.1:60156`
- Workspace root: `/Users/pedronauck/dev/qa-labs/agh-mem-v2-20260505-221648-724447-lab`
- Stable workspace identity: `01KQX3HAT21RHVV7MWA104FEJG`

## Reproduction

1. Invoke `agh__memory_propose` and `agh__memory_note` against the same workspace while the daemon is running.
2. Inspect the `agh__memory_note` decision payload.

## Expected

The controller target set contains only committed memory documents. Transient files created by atomic write implementation details are not visible to scan, recall, target selection, or public payloads.

## Actual

`agh__memory_note` returned a rules-only NOOP because the controller observed multiple plausible targets, including an atomic temp file:

`workspace::01KQX3HAT21RHVV7MWA104FEJG::project_qa-native-propose.md.tmp-838982806`

The temp file was not durable after the write completed, but it was visible long enough for the concurrent controller decision to consider it.

## Evidence

- Native propose payload: `.compozy/tasks/mem-v2/qa/logs/TC-INT-003/native-tool-propose-stable.json`
- Native note payload with temp target: `.compozy/tasks/mem-v2/qa/logs/TC-INT-003/native-tool-note-stable.json`
- Post-write temp scan: `.compozy/tasks/mem-v2/qa/logs/TC-INT-003/tmp-artifact-scan-after-parallel-native-writes.txt`

## Root Cause

`Store.Scan` filtered `MEMORY.md` and dotfiles, but not atomic write temp filenames. `fileutil.AtomicWriteFile` creates files using `<target>.tmp-*` in the same directory before rename, so concurrent scans could parse those temporary markdown-shaped files as committed memory documents.

## Fix

`internal/memory.Store` now skips filenames containing `.tmp-` during scan candidate discovery. `internal/memory/store_test.go` covers both `Store.Scan` and `Store.ListTargets` to prevent temp files from becoming prompt-visible or controller-visible targets.

## Fix Evidence

- Focused test: `go test ./internal/memory -run 'TestStoreScanSkipsAtomicTempFiles|TestStoreScanReturnsNewestFirst' -count=1`
- Production fix: `internal/memory/store.go`
- Regression test: `internal/memory/store_test.go`
