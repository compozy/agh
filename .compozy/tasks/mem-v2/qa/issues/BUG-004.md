# BUG-004: Recall observability events had empty summaries

## Status

Fixed

## Severity

P1 / High

## Scenario

TC-INT-003 and TC-SEC-001: Memory recall observability must be useful to operators while remaining redaction-safe. The event summary should identify query/result shape without exposing raw decision bodies, prompts, or secrets.

## Environment

- QA lab: `/Users/pedronauck/dev/qa-labs/agh-mem-v2-final-20260505-223124-817067-lab`
- Manifest: `/Users/pedronauck/dev/qa-labs/agh-mem-v2-final-20260505-223124-817067-lab/qa-artifacts/qa/bootstrap-manifest.json`
- `AGH_HOME`: `/var/folders/7x/xg204hnd04b81fczcxvjlhzr0000gn/T/aghqa-1e7a3857d148/runtime`
- HTTP base URL: `http://127.0.0.1:62376`

## Reproduction

1. Execute a Memory recall/search through CLI, HTTP, or UDS.
2. Query `agh observe events --type memory.recall.executed --last 5 -o json`.
3. Run the Memory catalog/search runtime E2E.

## Expected

Each `memory.recall.executed` event has a concise, redaction-safe summary such as `query="search visibility sentinel postfix" results=1`, enabling operators to correlate recall behavior without inspecting raw database rows.

## Actual

`memory.recall.executed` events were emitted durably, but their summary field was empty in the observer read model. This made the event stream materially less useful during QA and failed the Memory runtime E2E observability assertion.

## Evidence

- Pre-fix runtime E2E failure: `go test -race -tags integration ./internal/daemon -run 'TestDaemonE2EMemoryCatalogCLIHTTPParityAndLegacyPathIsolation' -count=1`
- Event rows existed in SQLite, but the observer payload did not expose a useful summary before the fix.

## Root Cause

`internal/memory/recall_source.go` recorded recall events without setting the summary field consumed by the observability read surface.

## Fix

`RecordRecallExecuted` now writes a redaction-safe summary containing the query string and result count. It does not include recall bodies, decision traces, or prompt content.

## Fix Evidence

- Runtime E2E rerun: `go test -race -tags integration ./internal/daemon -run 'TestDaemonE2EMemoryCatalogCLIHTTPParityAndLegacyPathIsolation|TestDaemonE2EMemoryRecallUsesCatalogSynthesisWithoutMutatingStoredUserMessage' -count=1`
- Fresh post-fix observe payload: `.compozy/tasks/mem-v2/qa/logs/final-postfix/observe-memory-recall-executed.json`
- Summary assertion: `.compozy/tasks/mem-v2/qa/logs/final-postfix/assert-observe-recall-summary.txt`
