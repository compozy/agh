---
title: Agents
description: How AGH resolves agent definitions, advertises capabilities, launches providers, and supervises ACP subprocesses.
---

Agents are the executable unit AGH manages. A definition file declares the agent identity, the
provider that runs it, the permissions it inherits, and the capabilities it advertises. At session
start AGH resolves that definition into an ACP subprocess, injects environment, negotiates the
session, and supervises it through stop.

<OperatorNote>
  This section is the runtime model for agents. The `AGENT.md` file format itself lives under
  [Configuration → AGENT.md](/runtime/core/configuration/agent-md); reach for it when authoring, not
  when troubleshooting runtime resolution.
</OperatorNote>
