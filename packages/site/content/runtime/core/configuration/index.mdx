title: Configuration

Configuration is where the runtime stops being generic and starts matching one machine, one
workspace, or one team's defaults. AGH reads global settings from `~/.agh/config.toml`, layers
workspace overrides on top, and then applies environment variables where the runtime explicitly
supports them.

Use the pages in this section when you need the exact format or precedence rules for:

- `config.toml` and workspace overlays
- `AGENT.md` and `SKILL.md` frontmatter
- `mcp.json` sidecars
- environment variables and path resolution
- the exact files AGH reads and writes at runtime
