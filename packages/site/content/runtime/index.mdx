AGH Runtime is the local-first control plane for real agent work. It lets you run Claude Code,
Codex, Gemini CLI, and other agent CLIs as durable sessions with replayable history,
workspace-aware memory, automation, and observability.

<OperatorNote>
  Runtime docs are organized by operator job, not by internal package boundary. Start in Core for
  the operating model, then use Sessions, CLI Reference, or API Reference when you need the exact
  live contract for one surface.
</OperatorNote>

## Working model

- **Core** explains how AGH behaves as a runtime you operate.
- **Sessions** explains the durable unit AGH actually runs, stores, resumes, and audits.
- **CLI Reference** tells you exactly what the command surface looks like.
- **API Reference** mirrors the machine-facing contract behind the same control plane.

The intended flow is simple: learn the model once, then use whichever runtime surface fits the job.
