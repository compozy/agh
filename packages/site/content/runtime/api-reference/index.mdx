The AGH daemon ships a single HTTP surface under `/api`. This reference is built from
`openapi/agh.json` at every site build, so each request and response schema below matches the
binary you are running.

<OperatorNote label="How this reference stays truthful">
  Every endpoint page is generated from `openapi/agh.json`, which is rewritten by `make codegen`
  from the Go HTTP contract. Drift is enforced by `make codegen-check` in CI, so what you read here
  matches the daemon you build from this checkout.
</OperatorNote>

The CLI and the HTTP API operate the same daemon state. Use the
[CLI reference](/runtime/cli-reference) for end-to-end examples; use the per-tag pages here when
you need request bodies, response schemas, status codes, or SSE event shapes.

## Recommended Entry Points

### Automating session work

1. [Sessions](/runtime/api-reference/sessions) — `POST /api/sessions`, `POST /api/workspaces/:workspace_id/sessions/:session_id/prompt`, `GET /api/workspaces/:workspace_id/sessions/:session_id/events`, `GET /api/workspaces/:workspace_id/sessions/:session_id/stream`, `POST /api/workspaces/:workspace_id/sessions/:session_id/attach`, `GET /api/workspaces/:workspace_id/sessions/:session_id/recap`.
2. [Agents](/runtime/api-reference/agents) — list and inspect resolved agent definitions.
3. [Memory](/runtime/api-reference/memory) — show, write, search, and run dream checks for persistent context.

### Building operational dashboards

1. [Observe](/runtime/api-reference/observe) — task dashboard and task inbox.
2. [Memory](/runtime/api-reference/memory) — health and redacted operation history.
3. [Bridges](/runtime/api-reference/bridges) — health snapshots and SSE updates.

### Integrating runtime configuration

1. [Settings](/runtime/api-reference/settings) — read and mutate daemon settings.
2. [Vault](/runtime/api-reference/vault) — list metadata, write new refs, delete refs (no plaintext ever leaves the daemon).
3. [Workspaces](/runtime/api-reference/workspaces) and [Skills](/runtime/api-reference/skills) — register and inspect operator surfaces.

### Managing extensible desired state

1. [Resources](/runtime/api-reference/resources) — read and mutate desired-state objects.
2. [Automation](/runtime/api-reference/automation) — jobs, triggers, runs.
3. [Extensions](/runtime/api-reference/extensions) — search marketplace sources, install, update,
   remove, enable, disable, inspect, and read provenance/trust reports.

<OperatorNote>
  Vault read responses contain only `ref`, `namespace`, `kind`, `present`, `created_at`, and
  `updated_at`. Write requests accept `secret_value`, but no HTTP, UDS, CLI, or web response ever
  returns that value. Use UDS for local agent automation when the daemon HTTP listener is bound
  outside loopback.
</OperatorNote>
