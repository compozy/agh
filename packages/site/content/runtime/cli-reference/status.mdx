## agh status

Show consolidated runtime status for operators and agents.

```bash
agh status [flags]
```

`agh status` reads the same payload as `GET /api/status` over HTTP and UDS. The
JSON output includes daemon process state, session totals, observer health,
memory, automation scheduler state, task queues, bridge health, provider auth,
MCP server availability, skill registry state, config apply state, and log-tail
availability.

### Examples

```bash
# Human-readable runtime summary
agh status

# Agent-readable runtime state
agh status --output json

# Inspect provider auth and MCP availability
agh status --output json | jq '{providers, mcp_servers, config, log_tail}'
```

### Options inherited from parent commands

```bash
      --json            Emit JSON output
  -o, --output string   Output format: human, json, jsonl, or toon (default "human")
```

## Output Formats

Every AGH command supports `-o, --output`:

- `human` for interactive terminal use
- `json` for scripts and other machine-readable consumers
- `jsonl` for wait or streaming commands that emit one JSON record per line
- `toon` for compact agent-readable summaries

```bash
agh status -o json
```
