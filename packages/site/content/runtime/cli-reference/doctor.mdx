## agh doctor

Run runtime diagnostics and return stable diagnostic items for operators and
agents.

```bash
agh doctor [flags]
```

`agh doctor` reads the same payload as `GET /api/doctor` over HTTP and UDS. Probe
failures are reported as diagnostic items, so agents can inspect `status`,
`summary`, and `items` without scraping terminal output.

### Examples

```bash
# Human-readable diagnostics
agh doctor

# Agent-readable diagnostic items
agh doctor --output json

# Run only provider and MCP diagnostics
agh doctor --only provider --only mcp --output json

# Omit OK diagnostics
agh doctor --quiet --output json
```

### Options

```bash
      --exclude strings   Exclude the named probe ids or categories
  -h, --help              help for doctor
      --only strings      Run only the named probe ids or categories
      --quiet             Omit OK diagnostics
```

### Options inherited from parent commands

```bash
      --json            Emit JSON output
  -o, --output string   Output format: human, json, jsonl, or toon (default "human")
```

## Output Formats

Every AGH command supports `-o, --output`:

- `human` for interactive terminal use
- `json` for scripts and other machine-readable consumers
- `jsonl` for wait or streaming commands that emit one JSON record per line
- `toon` for compact agent-readable summaries

```bash
agh doctor -o json
```
