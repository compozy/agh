## agh automation runs get

Show one automation run

```
agh automation runs get <id> [flags]
```

### Options

```
  -h, --help   help for get
```

### Options inherited from parent commands

```
  -o, --output string   Output format: human, json, or toon (default "human")
```

## Output Formats

Every AGH command supports `-o, --output`:

- `human` for interactive terminal use
- `json` for scripts and other machine-readable consumers
- `toon` for compact agent-readable summaries

Example:

```bash
agh automation runs get <id> -o json
```

For scheduler-created runs, inspect `fire_id` and `scheduled_at` to identify the claimed scheduled
fire. If the scheduler advanced the cursor but delivery failed, `delivery_error` and
`delivery_error_at` contain the delivery diagnostic while the normal run `error` field remains
reserved for execution lifecycle failures.
