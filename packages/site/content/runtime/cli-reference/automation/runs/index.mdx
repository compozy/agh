## agh automation runs

Inspect automation run history

```
agh automation runs [flags]
```

### Options

```
  -h, --help                help for runs
      --job-id string       Filter by automation job ID
      --last int            Show only the most recent N runs
      --since string        Show runs since an RFC3339 timestamp or relative duration
      --status string       Filter by run status
      --trigger-id string   Filter by automation trigger ID
      --until string        Show runs until an RFC3339 timestamp or relative duration
```

### Options inherited from parent commands

```
  -o, --output string   Output format: human, json, or toon (default "human")
```

## Output Formats

Every AGH command supports `-o, --output`:

- `human` for interactive terminal use
- `json` for scripts and other machine-readable consumers
- `toon` for compact agent-readable summaries

Example:

```bash
agh automation runs -o json
```

Scheduled runs include `fire_id` and `scheduled_at` for correlation with durable scheduler state.
Dispatch or transport failures are reported separately as `delivery_error` and `delivery_error_at`
so scheduler cursor state can still be inspected independently.
