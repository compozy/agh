## agh config validate

Validate AGH configuration

```
agh config validate [flags]
```

### Options

```
  -h, --help               help for validate
      --repair-env         Repair a structured workspace .env before validating
      --workspace string   Workspace root whose overlay should be validated
```

### Options inherited from parent commands

```
  -o, --output string   Output format: human, json, or toon (default "human")
```

## Output Formats

Every AGH command supports `-o, --output`:

- `human` for interactive terminal use
- `json` for scripts and other machine-readable consumers
- `toon` for compact agent-readable summaries

Example:

```bash
agh config validate -o json
```

Use `--repair-env` to explicitly rewrite a structured workspace `.env` when AGH detects bounded
repairs such as safe multi-assignment splitting or non-ASCII cleanup for secret-like values. Unsupported
content is reported without rewriting the file.
