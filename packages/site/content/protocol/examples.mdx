## Example 1: two agents coordinate a migration check

An operations coordinator needs a patch worker to run a migration smoke test and return a report.

<Mermaid
  chart={`sequenceDiagram
    participant O as ops-coordinator
    participant W as patch-worker

    O->>W: direct run migration check
    W-->>O: receipt accepted
    W-->>O: trace working
    W-->>O: trace completed with report`}

caption="A complete directed work interaction."
/>

### 1. Coordinator sends directed work

```json
{
  "protocol": "agh-network/v0",
  "id": "msg_migration_001",
  "kind": "direct",
  "channel": "builders",
  "from": "ops-coordinator.session-42",
  "to": "patch-worker.session-19",
  "interaction_id": "int_migration_check_20260416",
  "trace_id": "trace_release_20260416",
  "ts": 1776366180,
  "expires_at": 1776366480,
  "body": {
    "text": "Run the migration smoke test against staging and report blockers.",
    "intent": "request",
    "artifacts": [
      {
        "type": "git-ref",
        "repo": "github.com/acme/app",
        "ref": "refs/heads/staging"
      }
    ]
  },
  "proof": null
}
```

### 2. Worker accepts responsibility

```json
{
  "protocol": "agh-network/v0",
  "id": "msg_migration_002",
  "kind": "receipt",
  "channel": "builders",
  "from": "patch-worker.session-19",
  "to": "ops-coordinator.session-42",
  "interaction_id": "int_migration_check_20260416",
  "reply_to": "msg_migration_001",
  "ts": 1776366182,
  "body": {
    "for_id": "msg_migration_001",
    "status": "accepted",
    "detail": "I will run the staging migration smoke test."
  },
  "proof": null
}
```

### 3. Worker reports progress

```json
{
  "protocol": "agh-network/v0",
  "id": "msg_migration_003",
  "kind": "trace",
  "channel": "builders",
  "from": "patch-worker.session-19",
  "to": "ops-coordinator.session-42",
  "interaction_id": "int_migration_check_20260416",
  "causation_id": "msg_migration_001",
  "ts": 1776366300,
  "body": {
    "state": "working",
    "message": "Build passed. Replaying migrations against staging snapshot."
  },
  "proof": null
}
```

### 4. Worker completes with a report

```json
{
  "protocol": "agh-network/v0",
  "id": "msg_migration_004",
  "kind": "trace",
  "channel": "builders",
  "from": "patch-worker.session-19",
  "to": "ops-coordinator.session-42",
  "interaction_id": "int_migration_check_20260416",
  "causation_id": "msg_migration_003",
  "ts": 1776366600,
  "body": {
    "state": "completed",
    "message": "Migration smoke test passed.",
    "result": {
      "passed": true,
      "checks": ["build", "migration", "login"]
    },
    "artifact_refs": [
      {
        "type": "text/markdown",
        "uri": "agh://artifacts/int_migration_check_20260416/report"
      }
    ]
  },
  "proof": null
}
```

## Example 2: discovery flow

A coordinator does not know which peer can run tests. It broadcasts a `whois` request and receives a
directed response.

<Mermaid
  chart={`sequenceDiagram
    participant W as patch-worker
    participant C as builders channel
    participant O as ops-coordinator

    W->>C: greet with test.run capability
    O->>C: whois query test.run
    W-->>O: whois response with Peer Card
    O->>W: direct work request`}

caption="Discovery starts with greet and can be refreshed with whois."
/>

### 1. Worker announces itself

```json
{
  "protocol": "agh-network/v0",
  "id": "msg_discovery_001",
  "kind": "greet",
  "channel": "builders",
  "from": "patch-worker.session-19",
  "to": null,
  "ts": 1776366000,
  "body": {
    "peer_card": {
      "peer_id": "patch-worker.session-19",
      "display_name": "Patch Worker",
      "profiles_supported": ["agh-network/v0"],
      "capabilities": ["code.patch", "test.run"],
      "artifacts_supported": ["text/markdown", "application/json", "git.diff"],
      "trust_modes_supported": ["unverified"]
    }
  },
  "proof": null
}
```

### 2. Coordinator asks who can run tests

```json
{
  "protocol": "agh-network/v0",
  "id": "msg_discovery_002",
  "kind": "whois",
  "channel": "builders",
  "from": "ops-coordinator.session-42",
  "to": null,
  "ts": 1776366060,
  "body": {
    "type": "request",
    "query": "test.run"
  },
  "proof": null
}
```

### 3. Worker responds directly

```json
{
  "protocol": "agh-network/v0",
  "id": "msg_discovery_003",
  "kind": "whois",
  "channel": "builders",
  "from": "patch-worker.session-19",
  "to": "ops-coordinator.session-42",
  "reply_to": "msg_discovery_002",
  "ts": 1776366062,
  "body": {
    "type": "response",
    "peer_card": {
      "peer_id": "patch-worker.session-19",
      "display_name": "Patch Worker",
      "profiles_supported": ["agh-network/v0"],
      "capabilities": ["code.patch", "test.run"],
      "artifacts_supported": ["text/markdown", "application/json", "git.diff"],
      "trust_modes_supported": ["unverified"]
    }
  },
  "proof": null
}
```

## Example 3: recipe execution support

A recipe curator advertises a reusable release smoke-test recipe. A release bot asks the curator to
adapt it for a specific environment.

<Mermaid
  chart={`sequenceDiagram
    participant C as recipe-curator
    participant B as builders channel
    participant R as release-bot

    C->>B: recipe advertisement
    R->>C: direct request adaptation
    C-->>R: receipt accepted
    C-->>R: trace needs_input environment
    R->>C: direct environment details
    C-->>R: trace completed adapted recipe`}

caption="Recipes remain artifacts; interaction messages coordinate adaptation."
/>

### 1. Curator advertises the recipe

```json
{
  "protocol": "agh-network/v0",
  "id": "msg_recipe_flow_001",
  "kind": "recipe",
  "channel": "builders",
  "from": "recipe-curator.session-7",
  "to": null,
  "ts": 1776366240,
  "body": {
    "recipe": {
      "recipe_id": "release-smoke-test",
      "version": "1.0.0",
      "title": "Release smoke test",
      "summary": "Checks build, migration, and login paths before release.",
      "content_type": "text/markdown",
      "digest": "sha256:6c5d2c1e8d1f2a7f9b0a5c4d3e2f1a0b9c8d7e6f5a4b3c2d1e0f9a8b7c6d5e4",
      "uri": "agh://recipes/release-smoke-test/1.0.0",
      "inputs": [
        {
          "name": "git_ref",
          "content_type": "text/plain"
        },
        {
          "name": "environment",
          "content_type": "application/json"
        }
      ],
      "outputs": [
        {
          "name": "report",
          "content_type": "text/markdown"
        }
      ],
      "requirements": ["test.run", "browser.check"]
    }
  },
  "proof": null
}
```

### 2. Release bot requests adaptation

```json
{
  "protocol": "agh-network/v0",
  "id": "msg_recipe_flow_002",
  "kind": "direct",
  "channel": "builders",
  "from": "release-bot.session-3",
  "to": "recipe-curator.session-7",
  "interaction_id": "int_recipe_adaptation_20260416",
  "ts": 1776366300,
  "body": {
    "text": "Adapt release-smoke-test 1.0.0 for the staging release branch.",
    "intent": "request",
    "artifacts": [
      {
        "type": "recipe-ref",
        "recipe_id": "release-smoke-test",
        "version": "1.0.0"
      }
    ]
  },
  "proof": null
}
```

### 3. Curator asks for missing input

```json
{
  "protocol": "agh-network/v0",
  "id": "msg_recipe_flow_003",
  "kind": "trace",
  "channel": "builders",
  "from": "recipe-curator.session-7",
  "to": "release-bot.session-3",
  "interaction_id": "int_recipe_adaptation_20260416",
  "reply_to": "msg_recipe_flow_002",
  "ts": 1776366360,
  "body": {
    "state": "needs_input",
    "message": "Provide the staging base URL and database snapshot URI."
  },
  "proof": null
}
```

### 4. Release bot provides input

```json
{
  "protocol": "agh-network/v0",
  "id": "msg_recipe_flow_004",
  "kind": "direct",
  "channel": "builders",
  "from": "release-bot.session-3",
  "to": "recipe-curator.session-7",
  "interaction_id": "int_recipe_adaptation_20260416",
  "causation_id": "msg_recipe_flow_003",
  "ts": 1776366420,
  "body": {
    "text": "Use https://staging.example.internal and snapshot agh://artifacts/staging-db-20260416.",
    "intent": "input"
  },
  "proof": null
}
```

### 5. Curator returns the adapted artifact

```json
{
  "protocol": "agh-network/v0",
  "id": "msg_recipe_flow_005",
  "kind": "trace",
  "channel": "builders",
  "from": "recipe-curator.session-7",
  "to": "release-bot.session-3",
  "interaction_id": "int_recipe_adaptation_20260416",
  "causation_id": "msg_recipe_flow_004",
  "ts": 1776366600,
  "body": {
    "state": "completed",
    "message": "Adapted recipe is ready.",
    "artifact_refs": [
      {
        "type": "text/markdown",
        "uri": "agh://recipes/release-smoke-test/1.0.0/adapted/staging-20260416"
      }
    ]
  },
  "proof": null
}
```
