This page separates shipped AGH Runtime behavior from protocol profiles that are documented for
implementers and future compatibility.

<OperatorNote label="Read this before trust pages">
  The current AGH Runtime implementation is `agh-network/v2`. It preserves `proof` as opaque JSON
  and does not verify Ed25519 signatures. Pages that say v1, trust, or verified identity are
  protocol contracts for implementers, not current AGH Runtime behavior.
</OperatorNote>

## Current v2 runtime behavior

The shipped runtime validates and routes v2 envelopes. It supports:

- the six core message kinds
- channel-scoped peer discovery
- directed work through `surface:"direct"` on conversation-bearing messages
- admission and progress through `receipt` and `trace`
- capability advertisement and transfer
- duplicate, expiry, unsupported-kind, not-target, not-found, busy, and internal rejection reasons
- audit records for sent, received, delivered, ignored, and rejected network behavior

Use runtime docs when you operate AGH itself:

- [Network Overview](/runtime/core/network)

- [Delivery and Safety](/runtime/core/network/delivery-and-safety)

## Choosing the right page

- If you are operating AGH today, start with [Coordinate Agents over AGH Network](/runtime/guides/coordinate-agents-over-network).
- If you are implementing the v2 wire shape, start with [Minimal Sender](/protocol/guide/minimal-sender).
- If you are adding NATS to a peer, read [NATS Transport](/protocol/guide/nats-transport) after the minimal sender.
- If you are designing verified identities, read [Ed25519 + JCS](/protocol/ed25519-jcs) and [Signature Verification](/protocol/verification) as v1 profile docs.
- If you want to make a public compatibility claim, read [Conformance Levels](/protocol/conformance) and state which pieces you have tested.
