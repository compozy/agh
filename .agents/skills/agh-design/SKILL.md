The authoritative design system lives in two places — read both before producing anything:

1. `DESIGN.md` (repo root) — the canonical specification: visual theme, color tokens, surface ramp, hairlines, type, motion, ADRs, anti-patterns.
2. `packages/ui/src/tokens.css` — the canonical CSS token source consumed by the runtime UI, the marketing site, and the docs site.

Supporting sources to pull from when building:

- `packages/ui/src/components/` — shadcn-derived primitives (`button.tsx`, `card.tsx`, `dialog.tsx`, `popover.tsx`, `sidebar.tsx`, `tabs.tsx`, `table.tsx`, `select.tsx`, `field.tsx`, `tooltip.tsx`, `sonner.tsx`, etc.) plus AGH-custom variants under `packages/ui/src/components/custom/`.
- `web/` — runtime operator UI (React 19 + TanStack + Tailwind v4); use it to see how tokens compose in product surfaces.
- `packages/site/` — Fumadocs marketing landing + docs at `agh.network`; use it to see the editorial Playfair Display rhythm and `/runtime/*` + `/protocol/*` doc trees.

If creating visual artifacts (slides, mocks, throwaway prototypes, landing sections), produce static HTML files the user can open directly. Pull the actual tokens by importing or inlining values from `packages/ui/src/tokens.css` — never invent hex values. Reuse the JSX components from `packages/ui/src/components/` as building blocks when the artifact is React/TSX; for pure HTML, mirror their class structure and token usage.

If working on production code, edit inside `web/`, `packages/ui/`, or `packages/site/` and consume tokens via the existing CSS variables (`var(--canvas)`, `var(--accent)`, `var(--line)`, etc.) — never hardcode hex.

If invoked without other guidance, ask what the user wants to build (a landing section? a docs page? a slide? a product screen?), gather scope and variations, and act as an expert designer outputting HTML artifacts _or_ production code depending on the need.

**Rules that matter most for this brand (full reasoning in `DESIGN.md`):**

- **Dark mode only.** `color-scheme: dark` is hardcoded. Never render on a white background.
- **One accent: `#E8572A`** (`--accent`). Use sparingly — one accent target per viewport.
- **Warm-tinted neutral ramp**, never cool/bluish. Surface ramp: `--rail` `#0c0b0b` → `--canvas` `#131211` → `--canvas-soft` `#1a1918` → `--canvas-tint` `#1c1b1a` → `--elevated` `#232220`.
- **Translucent hairlines, not solid borders.** Use `--line`, `--line-soft`, `--line-strong` (rgba whites). The legacy solid `#3C3A39` divider is gone.
- **Two-token shadow vocabulary.** Only `--shadow-overlay` (modals) and `--highlight` (active rim) carry shadow. Cards, popovers, dropdowns, sticky headers, list rows stay flat with a 1 px ring on `--line-soft`. No drop shadows on content, no glassmorphism, no decorative texture.
- **Type stack:** Inter Variable for UI, JetBrains Mono for metadata + mono uppercase eyebrows (`--tracking-mono` = `0.06em`). Playfair Display reserved for `packages/site/` marketing hero only. NuixyberNext reserved for the `agh` wordmark only.
- **Eyebrow primitive is mandatory for every uppercase mono label.** Render through `<Eyebrow>` (`@agh/ui`) with `case="upper"`, `size` (`eyebrow` default = 11 px / `badge` = 10 px / `micro` = 9 px), and `tone` (`muted` default / `subtle` / `strong` / signal palette). Inlining `font-mono` + `uppercase` + arbitrary `text-[Npx]` / `tracking-[Nem]` values is forbidden — see `DESIGN.md` §3 and the `tokens.css` `--text-eyebrow` / `--text-badge` / `--text-micro` / `--tracking-mono` contract.
- **Signal colors are desaturated and tint-only.** Success `#5fbf85`, warning `#d6a647`, danger `#e0635a`, info `#8e8eb5`. Appear as low-alpha tints on chips/pills, never as solid banners.
- **No emoji. No bluish gradients. No colored-left-border cards** (except the one highlighted row in the comparison table).
- **Copy is operator-first and dry-confident** — engineer-to-engineer, no hype, no "we", no exclamation marks. See `COPY.md` for the full product-language spec.
- **Surface glaze tokens are mandatory.** Use `--row-hover`, `--row-selected`, `--surface-glaze`, `--bar-fill`, `--input-fill`, `--btn-default-fill`, `--btn-default-hover`, `--badge-fill`. Inline `bg-[rgba(255,255,255,…)]` literals are forbidden.
- **Owner / kind palettes route through `--avatar-{agent|human|system}-{0..N}-{bg,fg}` tokens.** Raw owner hex literals under `web/src/**` and `packages/ui/src/**` are forbidden.
- **Modal-scrim alpha is `0.55` and applies `var(--overlay-blur)` (3 px).** `backdrop-filter: blur(...)` is permitted only on `.dialog-scrim` / `.sheet-scrim`; every other surface stays blur-free §10.
- **`::selection` resolves to `var(--accent-tint)` (10 %).** `--overlay-selection` is deleted; do not reintroduce it.
- **Shadow vocabulary is two tokens only — `--shadow-overlay` (modals) and `--highlight` (active rim).** Inventing `--shadow-card` / `--shadow-pop` is forbidden §6.
- **`--neutral-tint` is pinned at `rgba(150, 150, 155, 0.06)`.** Drifting to cooler `(122, 122, 128, …)` channels is forbidden.
- **`--font-display` is an Inter alias reserved for `tokens.css` and explicit logo/wordmark slots.** Production code under `web/src/**` and `packages/ui/src/**` must not consume `var(--font-display)` for type-ramp purposes.
- **Tailwind `rounded-md` / `rounded-lg` / `rounded-xl` utilities are forbidden.** Resolve every radius through the token contract — `rounded-(--radius-xs)` / `rounded-(--radius)` / `rounded-md` / `rounded-lg` / `rounded-icon-well`. See `DESIGN.md` §14.
- **Inlining `font-mono uppercase tracking-mono` (or arbitrary `tracking-[Nem]`) tuples for label purposes is forbidden.** That combination IS the eyebrow contract; render through `<Eyebrow>` instead. See `DESIGN.md` §3.
- **Body baseline is pinned to Inter 13.5 px / line-height 1.5 / letter-spacing −0.006em / `font-feature-settings: "cv01", "ss03", "cv11"`.** Set once in the `tokens.css` body rule; never override per-page.
- **Mono identifiers in row contexts (task-row, kanban-card, inbox-item, run-row) render as bare mono text, never as `<Pill mono>`.**
- **Lucide icons use `strokeWidth={1.75}` by default; only at the 11 px `xs` size does `strokeWidth={2}` apply.** The size ramp is 11 / 12 / 14 / 16 px (`xs` / `sm` / `default` / `lg`); use the unifying `<Icon>` helper from `@agh/ui`.
- **`Loader2` from `lucide-react` is banned in production routes (`web/src/**`).** Use `<Spinner>` for "this surface is loading data" and `.dot--pulse` for "this entity is in an active running state".
- **Shell grid is `56 px rail + 244 px sidebar + 1 fr content`.** Sidebar collapses to 220 px at viewport ≤ 1100 px and to a drawer overlay at ≤ 880 px via `useSidebarViewport()` (returns `"default" | "md" | "drawer"`). Never inline panel widths or breakpoint literals.
- **Active workspace nub + selected nav rail + selected list-row rail all paint `var(--fg-strong)`** (white, 2 × 16 px). Accent stays reserved for the active CTA, the rail logo plate, and the active rail brand identity — never the active row, never the active tab indicator.
- **`<Topbar>` is the canonical route-identity surface.** Use the slot vocabulary `{ title, trailing, count, back, backLabel, meta, overflow }`; the count chip flows automatically from `useNavCounts()` when the route declares `topbar: { navCountKey: "tasks" | "jobs" | … }`. `@agh/ui` MUST NOT call `useNavCounts()` directly inside `<Topbar>` — `useTopbarShellModel()` resolves the count and passes it as a prop.
- **Detail-mode topbar swaps to `back + title + id-chip + sep + meta + spacer + actions + overflow`.** The back slot is a 20 × 20 ghost chevron; back behaviour resolves through `router.history.back()` with parent-route fallback. Detail surfaces emit `<DetailHeader>` (24 px H1, 6-row anatomy) in addition to the topbar swap.
- **`<RuntimeConnectionIndicator>` is the single owner of the daemon LED.** No rail-bottom LED, no per-row LEDs. The footer dot encodes `connected + degraded` via the tone-and-pulse rule (`success` solid / `success` pulse / `danger` solid). The degraded heuristic flips when ANY `useNavCounts()` entry carries `stale: true`.
- **`<Button>` runtime variants are additive: `default | primary | outline | secondary | ghost | destructive | success | link | neutral` (9 total).** `primary` is an initial-parity alias for `default` (accent CTA); `neutral` paints `var(--btn-default-fill)` → `hover:var(--btn-default-hover)`. Border-color hover is banned everywhere; `<Button variant="outline">` is reserved for legacy callsites and forbidden in new code.
- **`<Pill>` ships 4 px radius across every size (`xs` / `sm` / `md` → 17 / 19 / 22 px height).** Neutral pill is borderless tint-only. Mono variants emit `font-semibold tracking-[0]` 10.5 px regardless of size. The `uppercase` prop is gone — callers that need uppercase emit `className="uppercase"` themselves or compose `<Eyebrow>`.
- **`<PillGroup>` track is borderless `var(--canvas-soft)` + `var(--radius-md)` (8 px), 2 px padding, 1 px gap.** Segments render Inter sentence-case 12 px / 510 / -0.005em (never `font-mono uppercase`); active state lifts via `var(--elevated)` + `box-shadow: var(--highlight)`. Count badge is 3 px radius literal on `var(--badge-fill)` / `var(--muted)` with `tabular-nums` — never `var(--accent)`.
- **`<Tabs variant="line">` underline is `var(--fg-strong)` 1.5 px (NOT `var(--accent)` 2 px).** Active count chip is `bg-[rgba(255,255,255,0.07)]` / `var(--fg)` — never accent. `<Tabs variant="lane">` ships the `·`-separated lane-tab pattern with bare mono 10.5 px / `var(--faint)` counts (no chip wrapper). The chipped `variant="default"` is deprecated.
- **`<Empty>` icon-well is borderless `var(--canvas-soft)` / `var(--radius-lg)` (10 px).** Layout is top-padded (64 / 0 / 0), not flex-centered. Title 18 / 510 / -0.022em via `--text-empty-h1`. Description max-width 54ch. CTA `margin-top: 22 px`.
- **`<Metric>` and `<KpiCard>` are flat (no border).** Labels are sentence-case Inter (NOT uppercase eyebrow); metric value 22 px via `--text-metric-value`, KPI value 28 px via `--text-kpi-value`. `<KpiCard>` is the canonical export.
- **`<Dialog>` title is 13.5 px / 510 / -0.012em via `--text-modal-title`.** Header padding 13/18, footer 11/18 on `var(--canvas-soft)`, body 18 px. Close button 26 × 26 ghost. Scrim resolves `var(--overlay-scrim)` (0.55) + `var(--overlay-blur)` (3 px). Modal width ladder is `--modal-w-sm / --md / --lg` (560 / 720 / 880 px) — never inline `sm:max-w-*`.
- **`<RadioCard>` selected state paints `var(--surface-glaze)` + `box-shadow: inset 0 0 0 1px var(--line-strong)`.** Never `border-(--accent)` or `bg-(--accent-tint)`. Padding `9 / 11`, radius `var(--radius)` (6 px). Hover state lifts to `var(--elevated)`.
- **`<CatalogCard>` is flat, padding 16 px, hover `var(--elevated)`.** Logo well is 24 × 24 (default) or 40 × 40 (`logoSize="lg"`) with `var(--surface-glaze)` background, `var(--muted)` glyph. Selected state is inset `var(--line-strong)` ring + `var(--surface-glaze)` — never accent border, never accent tint.
- **Form inputs ship `bg-(--input-fill)` + transparent ring at rest** (12.5 px / -0.005em, 8/11 padding, 32 px height). Focus paints `bg-(--canvas)` + `box-shadow: 0 0 0 1px var(--line-strong)`. `<Textarea variant="mono">` opts into 12 px JetBrains Mono. Inputs MUST NOT override height through `h-9` / `h-10` inside form modals.
- **Use `<DetailHeader>` for the detail hero — never inline a 24 px H1.** The primitive renders the 6-row anatomy (crumbs / pre-title / 24 px H1 via `--text-detail-h1` / pills / meta / actions); `back?: () => void` mounts the 20 × 20 ghost chevron. The 22 px page-head tier is gone from the runtime; `<Section>` H2 is 13 px and never a page-level H1.
- **Use `<FormSection>` (not `<Section>`) inside modals and editable forms.** Container is `var(--canvas-soft)` / `var(--radius-lg)` / 18-20 padding, no border. Head is 13 / 510 / -0.008em via `--text-section-head`; optional leading icon + right-aligned eyebrow slot. Row gap 6 px, between-row spacing 14 px. Form-row hints sit inline with the label, never below the input.
- **Use `<MonoId>` for identifiers in row contexts** (task-row, kanban-card, inbox-item, run-row, breadcrumbs, agent IDs, run IDs). The primitive renders bare lowercased mono 10.5 px via `--text-mono-id` / `--tracking-mono-id` (0) with optional inline copy slot — pass the raw identifier, the primitive lowercases at render time. Identifier `<Pill mono>` is banned.
- **Use `<RunCard>` for active-run + standalone run-detail rendering.** Top row pill stack (status pill + `<MonoId>` run-id + session info + attempt + optional warning), body 4-col grid `CHANNEL / QUEUED / STARTED / ELAPSED` with `<Time mode="relative">` cells. Container `var(--canvas-soft)` / `var(--radius-lg)` / 14/16 padding, no border, no `border-l-* border-l-(--accent)` rail anywhere.
- **Use `<DescriptionCard>` for markdown bodies.** Renders through `STREAMDOWN_SAFE_CONFIG` (5 defenses: `skipHtml`, 13-element `disallowedElements`, URL-scheme allowlist via `defaultUrlTransform`, no streamdown chrome, canonical HTML overrides + `SafeImage` external-URL fallback). The XSS regression corpus in `description-card.test.tsx` is the security gate — do NOT relax it without an ADR.
- **Use `<OwnerAvatar>` for owner identity surfaces** (kanban cards, task rows, agent cards, timeline events, run cards). Background / foreground resolve via `colorsFor(ownerKind, ownerId)` against the tokenised `--avatar-{agent|human|system}-N-{bg,fg}` palette; sizes `sm` (20) / `default` (24) / `lg` (32); 2-character monogram by default with optional `glyph` slot. Aria-label is auto-emitted as `"{Role} {Name}"`.
- **Motion uses exactly three named keyframes — `dot--pulse` (1.8 s accent-glow), `pill-pulse` (2.2 s currentColor), `pulse-accent` (2 s accent-tint).** Tailwind's generic `animate-pulse` is banned across `web/src/**` and `packages/ui/src/**`. Duration tokens are `--dur` (140 ms) + `--dur-slow` (200 ms) + `--ease`; aliases (`--ease-in-out` / `--duration-fast/base/slow` / `--ease-out`) are dropped.
- **Status → tone is one declarative typed dictionary at `web/src/lib/status-tone.ts`** (split into `TASK_STATUS_TONE` / `RUN_STATUS_TONE` / `TASK_LANE_TONE`). `in_progress` is `info`, never `accent`; `completed` is `success`. Consumers MUST import from the dictionary — no inlined switch statements, no per-system tone maps.
- **`<PriorityBars>` always renders three bars (4 / 8 / 12 px tall).** Level changes color of all three bars, never bar count: `urgent → var(--danger)`, `high → var(--warning)`, `medium → var(--fg)`, `low → var(--faint)`. The `tone` prop is dropped — color derives from `level` only.
- **`<QueueHealthSparkline>` consumes `recharts` via the `@agh/ui` adapter.** Default bar fill `var(--bar-fill)`; cells with `stuck: true` switch to `var(--accent-tint-strong)`. Tooltip chrome resolves through `var(--canvas-soft)` / `var(--line)` / `var(--fg-strong)`; never paint solid `var(--accent)` on a bar.
- **`<StatusDot>` ships 4 tones (`warning | danger | accent | faint`) × 2 modes (`solid | ring`) × 2 sizes (6 / 5 px).** Defaults to `aria-hidden="true"` unless a `label` prop is provided.
- **`<Time>` is the canonical timestamp primitive** (30 s tick refresh, accepts `mode: "relative" | "absolute"`, emits `<time dateTime title>`). Helpers (`formatRelativeTime` / `formatAbsoluteTime` / `formatDuration` / `FORMAT_TIME_FALLBACK`) live in `@agh/ui` at `packages/ui/src/lib/format-time.ts`; `web/src/lib/format-time.ts` is a thin re-export shim — never duplicate the implementation.
- **Anti-pattern (compile-fail): `font-mono uppercase tracking-mono` (or `tracking-[Nem]`) inline tuple.** That combination IS the `<Eyebrow>` contract; reinventing it skips the primitive and the `--text-eyebrow` / `--tracking-mono` tokens. Flagged by `compozy-design-system/no-inline-eyebrow` + `no-inline-design-tuples`.
- **Anti-pattern (compile-fail): `text-[22px] font-medium tracking-[-0.026em]` page-h1 tuple inline.** The 22 px tier no longer exists in the runtime — page identity lives in `<Topbar>`, detail surfaces use `<DetailHeader>` (24 px). Flagged by `compozy-design-system/no-inline-design-tuples`.
- **Anti-pattern (compile-fail): `border-l-* border-l-(--accent)` / `border-l-accent` on cards or rows.** Active surfaces use the `var(--fg-strong)` indicator rail. Flagged by `compozy-design-system/no-side-stripe-accent`.
- **Anti-pattern (compile-fail): `hover:border-(--accent)` / `hover:bg-(--accent)` / `hover:text-(--accent)`.** Hover changes background tint via `--hover` / `--row-hover` — never border / text color. Accent is the resting active/CTA state only. See `DESIGN.md` §10 "Don't trigger accent on hover".
- **Anti-pattern (compile-fail): `bg-[rgba(255,255,255,0.0NN)]` literals.** Resolve through `--row-hover` / `--row-selected` / `--surface-glaze` / `--bar-fill` / `--input-fill` / `--btn-default-fill` / `--btn-default-hover` / `--badge-fill`. Flagged by `compozy-design-system/no-design-glaze-rgba`.
- **Anti-pattern (compile-fail): `import { Loader2 } from "lucide-react"` under `web/src/routes/**`.** Use `<Spinner>` for "this surface is loading data" and `.dot--pulse` for "this entity is in an active running state". Flagged by `compozy-design-system/no-banned-imports`. See `DESIGN.md` §10 "Don't import Loader2".
- **Anti-pattern (review fail): `<Pill tone="accent">` co-located with `<Button variant="primary">` in the same toolbar / topbar / surface.** Accent budget is one target per viewport — promote the CTA, demote the pill (default neutral or `--badge-fill` chip). Caught at PR review by combined class-list snapshots on toolbar regions.

**Site profile (when working in `packages/site/`):**

Color, surface, text, accent, signal, motion, hairlines, overlays, eyebrow utilities, and shadow whitelist are SHARED with runtime — read `packages/ui/src/tokens.css` and DESIGN.md §1–§12. Site only EXTENDS the contract for typography clamps, layout widths, marketing/doc body styles, and bento overlays. The full extension table lives in **DESIGN.md §13 "Site Profile"**.

When generating site UI:

- Use the `--text-site-*` clamp utilities (`text-site-hero`, `text-site-blog-title`, `text-site-bento-2xl`, `text-site-doc-heading`, …) — declared in `packages/site/app/global.css @theme inline`. Do not reach for arbitrary `text-[clamp(...)]`; if a needed clamp is missing, add it to DESIGN.md §13 + `app/global.css` in the same change.
- Layout widths: `max-w-(--site-layout-width)` (1200px landing), `max-w-(--site-doc-layout-width)` (96rem docs), `w-(--site-doc-toc-width)` (14rem TOC). Never inline literal `max-w-[1200px]`.
- Playfair Display via `--font-display` is allowed only inside `.site-home` (landing hero + section H2). NuixyberNext via `--font-wordmark` is allowed only on the literal `agh` wordmark. Anywhere else (blog, changelog, runtime/protocol docs, MDX bodies) uses Inter Variable.
- Doc article copy goes through the `.site-doc-body` class (line-height 1.8, max-width 72ch on prose blocks, h2 with top-border separator). Bento illustrations dim into `--rail` via `.site-bento-overlay-{runtime,network,bridges,memory,extensibility}`.
- Inventing a new `--color-*` or non-clamp `--text-*` token in site-scope drifts from the contract. Add it to DESIGN.md §13 first, then implement.

When generating runtime UI (`web/` or `packages/ui/`): never reach for Playfair, NuixyberNext, the `--text-site-*` clamps, or the `.site-*` classes. Stay on the runtime Inter-510 ladder defined in DESIGN.md §3.
